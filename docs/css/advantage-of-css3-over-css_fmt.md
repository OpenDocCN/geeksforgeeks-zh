# CSS3 相对于 CSS 的优势

> 原文:[https://www.geeksforgeeks.org/advantage-of-css3-over-css/](https://www.geeksforgeeks.org/advantage-of-css3-over-css/)

`CSS3` 是 `CSS` 的最新版本。它比它的前身有许多优点。`CSS3` 在 2010 年初开始流行。当时，一些浏览器甚至不支持 `CSS3`，因此网络开发人员社区需要一段时间才能开始在他们的项目中实现这一点。在这里，我们将看到 `CSS3` 相对于 `CSS` 的优势，也将讨论为什么 `CSS3` 在短时间内获得如此巨大的人气。

## Responsiveness
`CSS3` 自然支持响应式设计，并且能够处理媒体查询。媒体查询有助于使网站能够响应，因为我们可以为不同的屏幕宽度应用个性化的 `CSS` 属性。因此，网站在访问的每个设备上看起来都很好。

## Color
添加了新的颜色格式，如 `RGBA`（红、绿、蓝、Alpha）、`HSL`（色调、饱和度、亮度）和 `HSLA`（色调、饱和度、亮度、Alpha）。这对网页设计师有帮助，因为它有助于他们为网页应用样式和不同的颜色效果。还添加了渐变和不透明度属性。

## Reduce alignment problems
盒子大小调整修复了一些令人烦恼的对齐问题，因为现在开发人员可以使用内边距、外边距和其他属性来为分区赋予适当的大小。它有助于创建复杂的网页结构。`CSS` 网格有助于创建响应式网页，而无需使用表格行和列。因此，对齐问题得到了解决，使网站对移动设备更友好。

## Animation
`CSS` 要求开发人员使用脚本语言来创建动画。另一方面，`CSS3` 引入了动画功能，如文本阴影。它减少了开发人员的工作量，因为在早期阶段应用这样的动画很困难，但随着这些新的 `CSS` 属性的引入，它变得容易得多。

## Compatibility
如前所述，`CSS3` 在发布时并未得到许多浏览器的支持，因此开发人员需要一些时间来掌握这一点。然而，现在每个现代浏览器都支持 `CSS3`，这就是为什么现在建立的每个网站都使用 `CSS3` 而不是普通的 `CSS`。

## JavaScript independence loads faster
如上所述，早期的开发人员必须混合使用 `JavaScript` 和 `CSS` 来提供甚至像文本阴影这样简单的装饰。随着 `CSS3` 的引入，`CSS` 对 `JS` 的依赖性降低了，从而减少了网页的加载时间。

## Testing
早期版本的 `CSS` 在测试部分有点复杂。然而，`CSS3` 被划分为更小的模块，这使得用户更容易运行兼容性测试并单独测试组件。因此，错误检测变得更容易，并为故障排除节省了大量时间和精力。

## Platform-friendly
`CSS3` 与所有平台或设备兼容，例如移动电话和平板电脑。

## Border radius
`CSS` 的 `border-radius` 属性有助于使边框和图像 `div` 元素的形状变圆，这过去需要大量的格式化和 `Photoshop` 工作。另一个大问题是它对圆角图像的支持。还添加了其他属性，如阴影。

因此，这就是 `CSS3` 的突然流行背后的原因，以及 `CSS3` 相对于 `CSS` 的优势。
