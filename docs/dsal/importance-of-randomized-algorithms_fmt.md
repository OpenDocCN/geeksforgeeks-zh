# 随机化算法的重要性

> 原文：[https://www.geeksforgeeks.org/importance-of-randomized-algorithms/](https://www.geeksforgeeks.org/importance-of-randomized-algorithms/)

## 简介

*   [Randomization](https://www.geeksforgeeks.org/randomized-algorithms/) 是一个重要的概念，因此随机化[算法](https://www.geeksforgeeks.org/fundamentals-of-algorithms/)被用于许多领域，例如[数论](https://www.geeksforgeeks.org/number-theory-competitive-programming/)、[计算几何](https://www.geeksforgeeks.org/geometric-algorithms/)、[图论](https://www.geeksforgeeks.org/graph-data-structure-and-algorithms/)和分布式计算。
*   随机化算法的输入与确定性算法的输入类似，但有一系列随机位可供算法用于随机选择。
*   换句话说，随机化算法是一种其行为依赖于输入的算法。与确定性算法类似，随机选择是其逻辑的一部分。
*   因此，即使对于相同的输入，算法也会给出不同的输出。
*   换句话说，该算法是随机的；因此，其运行时间通常用随机变量来解释。

## 优势

*   随机化算法以其简单性而闻名。
*   任何确定性算法都可以轻松转换为随机化算法。这些算法易于理解和实现。
*   随机化算法非常高效。
*   与任何确定性算法相比，它们几乎不占用执行时间和空间。
*   与确定性算法相比，随机化算法表现出更好的渐近界限。
*   换句话说，随机化算法的算法复杂度优于大多数确定性算法。
*   可靠性是许多关键应用中的一个重要问题，因为并非所有随机化算法都能给出正确答案。
*   此外，许多随机化算法可能不会终止。
*   因此，可靠性是一个需要处理的重要问题。
*   随机化算法的质量取决于作为算法一部分的随机数生成器的质量。
*   与其他设计范式不同，随机化算法不使用单一的设计原则。
*   相反，人们应该将随机化算法视为使用一组原则设计的算法。

以下小节列出了一些设计原则：

### 证人概念

*   这个原则涉及检查给定输入是否具有属性 X 的问题。
*   它是通过找到一个被称为证人或证书的特定对象来建立的。
*   证人被识别以证明输入确实具有期望的属性 X 的事实。
*   通过进行更少的试验，可以发现属性是否真的存在。
*   证人的存在是基于证人不存在时属性 X 不成立的有力证明。这个原理用素性检验的例子来说明。

### 指纹

*   根据定义，指纹是代表较大对象的简短信息。
*   指纹识别是一种仅通过比较它们的简短指纹来比较两个大对象的技术。
*   如果两个指纹不匹配，那么对象 A 和 B 是不同的。
*   然而，如果指纹匹配，则有强有力的间接证据表明两个对象是相同的。

### 检查恒等式

*   假设给定了一个代数表达式。问题是要检查表达式的求值结果是否为零。
*   检查恒等式的原理是将随机变量代入给定的代数方程，并检查表达式的求值结果是否为零。
*   如果不为零，则给定的表达式不是恒等式。
*   否则，有强有力的间接证据表明该表达式恒等于零。

### 随机采样和排序

*   算法的性能有时可以通过随机化输入分布或顺序来提高。
*   可以观察到，对于某些类型的输入，算法的性能可能更高或只是可以接受。
*   在这里，随机化导致随机排序、划分和采样。
*   此外，随机化算法使用随机样本来收集有关输入分布的信息。招聘问题说明了这一点。

### 挫败对手

*   随机化算法可以被视为一个人与对手之间的游戏，即提出算法的人与试图通过设计适当的输入来击败算法的对手之间的游戏，这使得算法需要更长的时间。
*   换句话说，随机化算法可以被视为从一大组确定性算法中选择一个算法，并且这种选择可以被视为通过给予随机输入使事情变得更加困难，从而使任务更加困难。
