# 库克-莱文定理或库克定理

> 原文：[https://www.geeksforgeeks.org/cook-levin-theorem-or-cooks-theorem/](https://www.geeksforgeeks.org/cook-levin-theorem-or-cooks-theorem/)

## 定理概述

在计算复杂性理论中，库克-莱文定理，也称为库克定理，指出布尔可满足性问题是 NP 完全的。也就是说，它在 NP 中，NP 中的任何问题都可以通过确定性图灵机在多项式时间内简化为布尔可满足性问题。

## 历史证明

斯蒂芬·阿瑟·库克和 L.A. 莱文在 1973 年独立证明了可满足性问题（`SAT`）是 NP 完全的。斯蒂芬·库克（Stephen Cook）在 1971 年发表了一篇题为《定理证明程序的复杂性》的重要论文，他在论文中概述了通过将 NP 完全问题简化为 `SAT` 来获得其证明的方法。他证明了电路-SAT 和 3nf-SAT 的问题和 `SAT` 一样难。同样，列昂尼德·列文在当时的苏联独立研究这个问题。`SAT` 是 NP-complete 的证明，就是在这两位科学家的努力下获得的。后来，卡普把哈密顿圈、顶点覆盖和团等 21 个优化问题简化到 `SAT` 上，证明了这些问题是 NP 完全的。

## SAT 问题定义

因此，`SAT` 是一个重大问题，可以表述如下：

给定一个布尔表达式 `F`，具有 `n` 个变量 `x1`，`x2`，…，`xn`，和布尔运算符，有没有可能对变量赋值为真或假，使得二进制表达式 `F` 为真？

这个问题也被称为公式-SAT。`SAT`（公式-SAT 或简称 `SAT`）取一个布尔表达式 `F`，并检查给定的表达式（或公式）是否可满足。如果求值结果为真，则布尔表达式对于某些有效的变量赋值是令人满意的。在讨论 `SAT` 的细节之前，现在让我们讨论布尔表达式的一些重要术语。

## 重要术语

- **Boolean variable**：一个变量，例如 `x`，只能有两个值，真或假，这被称为布尔变量。
- **Literal**：一个文字可以是一个逻辑变量，例如 `x`，或者是它的否定，即 `x` 或 `¬x`；`x` 称为正文字，`¬x` 称为负文字。
- **Clause**：一系列变量 `(x1, x2, ... , xn)` 可以通过逻辑 `或`（`∨`）运算符分隔，这被称为子句。例如，`(x1 ∨ x2 ∨ x3)` 是一个包含三个文字的子句。
- **Expression**：你可以使用布尔运算符将所有前面的子句组合成一个表达式。
- **CNF form**：如果子句集由 `与`（`∧`）运算符分隔，而文字由 `或`（`∨`）运算符连接，则该表达式为 CNF 形式（合取范式）。以下是 CNF 表达式的示例：
    ```
    F = (x1 ∨ x2 ∨ x3) ∧ (¬x1 ∨ x3)
    ```

## SAT 的 NP 完全性

因此，`SAT` 是最棘手的问题之一，因为除了暴力方法之外，没有其他已知的算法。强力算法将是指数时间算法，因为需要尝试可能的赋值来检查给定的布尔表达式是否为真。斯蒂芬·库克和列昂尼德·莱文证明了 `SAT` 是 NP 完全的。

库克向 `SAT` 证明了其他难题的减少。卡普通过使用卡普约简将其简化为 `SAT`，提供了 21 个重要问题的证明，如哈密顿圈、顶点覆盖和团。

## 三种 SAT 变体

让我们简单介绍一下三种 `SAT`。它们如下：

1.  **Circuit-SAT**：一个 Circuit-SAT 可以表述如下：给定一个布尔电路，它包含 `与`、`或`、`非`、`n` 个输入等。同样，这些问题的难度在于电路的 `n` 个输入。需要检查 `2^n` 种可能的输出。因此，这个暴力算法是指数算法，所以它是一个困难的问题。
2.  **CNF-SAT**：这个问题是 `SAT` 的一个限制性问题，这里的表达式应为合取范式。如果所有子句都由布尔 `与` 运算符连接，则该表达式被称为合取形式。与 `SAT` 的情况一样，这是关于为 `n` 个变量赋真值，使得表达式的输出为真。
3.  **3-CNF-SAT (3-SAT)**：这个问题是另一个变体，其中的附加限制是表达式为合取范式，并且每个子句应恰好包含三个文字。这个问题也是关于为布尔表达式的 `n` 个变量赋真值，使得表达式的输出为真。简单来说，给定一个 3-CNF 表达式，3-SAT 问题是检查给定的表达式是否可以满足。

这些问题可以用来证明一些重要问题的 NP 完全性。
