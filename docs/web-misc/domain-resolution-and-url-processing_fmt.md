# 域名解析和网址处理

> 原文：[https://www.geeksforgeeks.org/domain-resolution-and-url-processing/](https://www.geeksforgeeks.org/domain-resolution-and-url-processing/)

域名是用来访问任何网站的地址。域名本质上是独一无二的，非常容易记住。通过输入域名，用户可以到达他想要的任何网站。任何网站的实际地址都是非常复杂的，但是借助于域名，这个复杂的地址可以被转换成一个简单易懂的形式。域名是借助于域名系统的规则和程序形成的。

## 为什么域名很重要？

域名容易记住，但 IP 地址不容易记住。我们举个电话簿的例子，我们不记得人的电话号码，我们只记得人的名字，然后用他们的名字搜索号码，而不是用他们的电话号码。IP 地址可以在后端更改，但域名保持不变。

## 历史

自 ARPANET 时代以来，我们一直使用人类可理解的主机数字地址形式。以前，网络上的每台计算机都用于从 SRI 的计算机中检索主机文件，SRI 用于将计算机主机名映射到数字地址。此后，随着网络开始迅速发展，用传统方法来完成这项任务变得几乎不可能，因此在 1983 年，域名系统被引入阿帕网。

## 特点

*   域名越短，越容易记忆和理解。域名越长，越难记忆和理解。因此，建议保持域名简短明了。
*   域名应该易于拼写，因为太复杂的话用户很难清晰地记住。
*   在域名中`.com`扩展名是最受欢迎的，因为很多人记住带有此扩展名的网站，而经常忘记其他扩展名。
*   有时一个域名没有任何意义，但它变得如此流行，以至于人们只需要听到它的名字就能认出这个域名，所以我们也可以创建一个品牌，除非域名成为一个品牌，否则我们真的不需要任何意义。
*   最好不要在域名中包含连字符和数字，因为这非常难以记忆。

## 域名解析和网址处理步骤

当我们在网络浏览器中键入一个网址时，我们的计算机需要将它们转换成一个 IP 地址。这样它就可以联系我们的网络服务器并发送给您。这称为正向查找，因为我们正在将主机名转换为 IP 地址。当我们将一个 IP 地址转换为主机名时，这也被称为反向查找。

*   首先，我们将输入我们想要访问的网站名称。例如：`facebook.com`。
*   然后域名将被映射到它们对应的 IP 地址，解析器的责任就是将域名转换为 IP 地址。在请求和根服务器之间有一个解析器。
*   网站请求首先到达根服务器，这可以通过递归方法或迭代方法完成。世界上有 13 台根服务器。
*   然后解析器给解析器下一级的 IP 地址。下一级由通用服务器或国家服务器组成。
*   解析器从右向左解析域名，这意味着在我们的例子中，它将首先被解析`.com`。然后`.com`提供一个链接给解析器，接着我们的请求将被转发到权威服务器。
*   权威服务器存储所有的 IP 地址和名称。一个特定的网站可以有多个 IP 地址。Facebook 拥有超过 70,000 台服务器，因此可以有多个 IP 地址。
*   这个 IP 地址被交给解析器，然后解析器将这个 IP 地址交给我们的计算机。然后我们就可以访问计算机中的数据。

这些步骤只在我们第一次访问任何网站时执行。当我们第二次访问该网站时，我们的请求不会被转发到根服务器，因为我们的互联网服务提供商（ISP）缓存了该 IP 地址，因此无论何时您需要访问该网站，它都会轻松提供 IP 地址。

## 优势

*   好的域名可以增加可信度，并且也能与其他域名区分开来。
*   此外，域名能带来品牌认知度并吸引许多用户。
*   一旦域名在网民中流行起来，它可以提高搜索引擎的排名。
*   域名在网络空间中建立了独特的身份。
*   域名可以转售、租赁和交换，这也是一个巨大的优势。

## 缺点

*   DNS 注册只能由`ICANN`控制，这是一个巨大的挑战，因为它只与一个国家相关联，因此也挑战了网络中立性。
*   在 DNS 服务器上只能看到 IP 地址，从现在起它可能被黑客操纵。
*   此外，DNS 服务器基于主从关系，这是另一个挑战。
*   如果主机开始出现故障，就很难访问网页。
*   在许多情况下，黑客以服务器机器为目标，并将用户重定向到另一个网页，从而导致网络钓鱼。
