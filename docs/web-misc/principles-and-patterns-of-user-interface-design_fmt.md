# 用户界面设计的原则和模式

> Original: [https://www.geeksforgeeks.org/principles-and-patterns-of-user-interface-design/](https://www.geeksforgeeks.org/principles-and-patterns-of-user-interface-design/)

**用户界面设计**是一个可以看到用户与系统有效交互的平台。 用户更多地是通过界面而不是功能来判断系统。 设计得不好或不好的界面会导致用户不正确地使用系统。 业务系统中的许多用户使用图形用户界面与系统交互。 甚至还使用基于文本的界面。 因此，在设计用户界面时考虑原则和模式是非常重要的。

## 用户界面设计原则

在设计用户界面时应该牢记一些原则。 这些原则如下：

1.  `user familiarity`：不使用计算机术语，而是使用面向用户的术语。例如，在“Microsoft Office”软件中，使用诸如文档、电子表格、信件、文件夹等术语，而避免使用目录、文件和身份等术语。
2.  `consistency`：应在用户界面中保持适当的一致性。例如，命令或菜单的格式应该相同。
3.  `minimum accident`：命令应以已知方式运行。这使得用户更容易预测界面。例如，在文字处理文档的“工具”菜单下应该有拼写和语法检查。
4.  `recoverability`：系统应提供从错误中恢复的工具，以便用户可以纠正它们。例如，撤销可以纠正这些错误，或者在删除某些内容时必须提供确认操作，以便用户在删除前三思。
5.  `user Guide`：如果提供了一些用户指南，如帮助系统、在线手册等，新手用户可以有效地使用用户界面。

## 用户界面设计模式

用户界面设计模式在当今世界非常重要。 如今，在设计程序或软件时，包含用户界面设计模式是非常常见的，也是非常必要的。 这些设计模式有助于解决问题，因为该设计模式在许多情况下都可以作为问题的解决器。 下面的表格显示了一些例子：

| 图案 / 样板 / 样式 / 方式 | It means |
| --- | --- |
| `Progress indicator` | 在许多程序中，我们在安装过程中看到这种模式，以便用户可以感觉到屏幕背后正在进行一些处理或操作。它用于向用户保证系统在安装过程中不会挂起。 |
| `Guide` | 此模式通常用于通过各种窗口指示任务已完成。它用于指示用户安装已完成。 |
| `Shopping Cart` | 这通常用于在线购物网站，如亚马逊、Flipkart等。它用于为用户提供可购买的商品列表。 |
