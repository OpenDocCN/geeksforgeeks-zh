# 嵌入式系统分类

> 原文：[https://www.geeksforgeeks.org/classification-of-embedded-systems/](https://www.geeksforgeeks.org/classification-of-embedded-systems/)

**嵌入式系统**根据两个因素进行分类，即

1.  性能和功能要求
2.  微控制器的性能

## 根据性能和功能需求分类

根据性能和功能需求，分为以下 4 种类型：

### 1. 实时嵌入式系统

`Real-Time Embedded System` 是严格时间特定的，这意味着这些嵌入式系统在特定/定义的时间间隔内提供输出。这类嵌入式系统在危急情况下提供快速响应，优先处理基于时间的任务执行和输出生成。因此，实时嵌入式系统被用于国防部门、医疗保健部门以及其他一些工业应用中，其中及时输出至关重要。

此外，该实时嵌入式系统分为两种类型，即

*   **软实时嵌入式系统**：在这些类型的嵌入式系统中，时间/截止日期并没有得到严格遵守。如果任务的截止日期已过（意味着系统没有在规定的时间内给出结果），仍然接受结果或输出。
*   **硬实时嵌入式系统**：在这些类型的嵌入式系统中，严格遵守任务的时间/截止日期。任务必须在两个时间框架（定义的时间间隔）之间完成，否则结果/输出可能不被接受。

**示例：**

*   交通管理系统
*   国防部门的军事用途
*   卫生部门的医疗用途

### 2. 独立嵌入式系统

`Stand Alone Embedded Systems` 是独立的系统，可以自行工作，不依赖于主机系统。它接收数字或模拟形式的输入并提供输出。

**示例：**

*   MP3 播放器
*   微波炉
*   计算器

### 3. 网络嵌入式系统

`Networked Embedded Systems` 连接到有线或无线网络，以向连接的设备提供输出。它们通过网络与嵌入式 Web 服务器通信。

**示例：**

*   家庭安全系统
*   自动取款机
*   刷卡机

### 4. 移动嵌入式系统

`Mobile Embedded Systems` 体积小、易于使用且所需资源较少。它们是最受欢迎的嵌入式系统。从便携性角度来看，移动嵌入式系统也是最佳选择。

**示例：**

*   MP3 播放器
*   移动电话
*   数码相机

## 根据性能和微控制器分类

根据性能和微控制器分为以下 3 种：

### 1. 小规模嵌入式系统

`Small Scale Embedded Systems` 使用 8 位或 16 位微控制器设计。它们可以由电池供电。处理器使用的内存和处理速度资源非常有限。这些系统主要不作为独立系统运行，而是作为计算机系统的某个组件，但它们不进行计算，专用于特定任务。

### 2. 中规模嵌入式系统

`Medium Scale Embedded Systems` 使用 16 位或 32 位微控制器设计。这些中规模嵌入式系统比小规模嵌入式系统更快。这些系统中硬件和软件的集成很复杂。`Java`、`C`、`C++` 是用于开发中规模嵌入式系统的编程语言。不同类型的软件工具，如编译器、调试器、模拟器等，用于开发这类系统。

### 3. 复杂或复杂的嵌入式系统

复杂或复杂的嵌入式系统使用多个 32 位或 64 位微控制器进行设计。这些系统是为执行大规模复杂功能而开发的。这些系统具有很高的硬件和软件复杂性。我们使用硬件和软件组件来设计最终系统或硬件产品。
