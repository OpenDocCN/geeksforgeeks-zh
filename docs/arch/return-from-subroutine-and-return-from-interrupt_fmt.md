## 从子程序返回和从中断返回

> 原文：[https://www.geeksforgeeks.org/return-from-subroutine-and-return-from-interrupt/](https://www.geeksforgeeks.org/return-from-subroutine-and-return-from-interrupt/)

在本文中，我们将讨论从子程序返回和从中断返回，并详细解释，最后以子程序返回和从中断返回的区别结束。我们一个一个来讨论。

## 从子程序返回

*   从子程序返回是将代码的执行状态返回到调用子程序之前的状态。
*   通常，子程序由 `call` 指令调用。在每个 `CALL` 指令处，下一条指令的 `IP` 和 `CS` 会被压入堆栈，然后控制权转移给该过程。在程序结束时，必须执行 `ret` 指令。
*   `RET` 用于从之前由 `call` 调用的子程序返回。当它执行时，之前存储的内容（如 `IP` 和 `CS`）会从堆栈中取出并放入 `CS` 和 `IP` 寄存器，主程序的执行得以继续。
*   因此，可以认为调用子程序会将当前地址 `PC+2` 压入堆栈。从子程序返回将返回由堆栈顶部两个字节计算出的地址。
*   首先，从堆栈中弹出最高有效字节，然后是最低有效字节。`RET` 的地址从堆栈加载。`RET` 执行后，堆栈及其指针将相应修改。

## 从中断返回

*   从中断返回是将代码执行返回到中断之前的状态，同时将标志保存到标志寄存器中。
*   通常，当中断发生时，通过调用其特定的服务例程来处理。当要调用中断服务例程时，在将控制权转移给它之前，`IP`、`CS` 和标志会一起存储在堆栈上，以指示中断服务例程（`ISR`）执行完毕后继续执行的位置。因此，在 `ISR` 结束时，执行 `iret`，从堆栈中检索 `IP`、`CS` 和标志的值，然后主程序继续执行。
*   `IRET` 或 `reti` 用于从 `ISR` 返回。它首先从堆栈加载标志并恢复其先前的值，即它会启用标志中断。从堆栈加载标志后，它将从堆栈加载 `CS` 和 `IP` 值到寄存器中，主程序继续执行。使用 `Pushf` 和 `POPF` 指令分别将标志压入和弹出堆栈。从堆栈加载 `IRET` 地址并启用全局中断标志。`IRET` 执行后，堆栈及其指针相应修改。

## 从子程序返回和从中断返回的区别

1.  `IRET` 指令用于退出中断程序，而 `RET` 用于从子程序返回。
2.  `IRET` 与 `RET` 类似，区别在于 `RET` 只会弹出两个字节到 `PC`，而 `IRET` 会重置中断使能（`ien`）触发器并从堆栈弹出两个字节。
3.  为了告诉处理器中断处理已经结束，可以重置标志，并使用 `IRET` 指令，而不仅仅是 `RET`。
4.  如果在 `ISR` 外部执行，`IRET` 与 `RET` 功能相同。
