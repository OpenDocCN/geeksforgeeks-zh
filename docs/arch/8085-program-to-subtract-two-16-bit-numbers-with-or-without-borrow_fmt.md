# 8085 程序减去两个 16 位数字有无借用

> 原文: [https://www.geeksforgeeks.org/8085-program-to-subtract-two-16-bit-numbers-with-or-without-borrow/](https://www.geeksforgeeks.org/8085-program-to-subtract-two-16-bit-numbers-with-or-without-borrow/)

## 问题
在 8085 微处理器中编写汇编语言程序，减去两个 16 位数字。

## 假设
*   项目起始地址: `2000`
*   输入内存位置: `2050`、`2051`、`2052`、`2053`
*   输出内存位置: `2054`，`2055`

## 示例
```
INPUT:
       (2050H) = 19H
       (2051H) = 6AH 
       (2052H) = 15H 
       (2053H) = 5CH

OUTPUT:
        (2054H) = 04H
        (2055H) = OEH
```

结果:
因此我们减去了两个 16 位数字。

## 算法
1.  获取 16 位数字的 `L` 寄存器中的 `LSB` 和 `H` 寄存器中的 `MSB`。
2.  用 `DE` 寄存器交换 `HL` 寄存器的内容。
3.  再次获取 16 位数字的 `L` 寄存器中的最低有效位和 `H` 寄存器中的最高有效位。
4.  用 `E` 寄存器的内容减去 `L` 寄存器的内容。
5.  用 `D` 寄存器的内容减去 `H` 寄存器的内容，借用上一步。
6.  将结果存储在内存位置。

## 程序
| 存储地址 | 记忆术 | 评论 |
| --- | --- | --- |
| `2000` | `LHLD 2050` | 加载地址为 `2050` 的 `H-L` 对 |
| `2003` | `XCHG` | 用 `DE` 对交换 `HL` 对 |
| `2004` | `LHLD 2052` | 加载地址为 `2052` 的 `H-L` 对 |
| `2007` | `MVI C, 00` | `C = 00` |
| `2009` | `MOV A, E` | `A = E` |
| `200A` | `SUB L` | `A = A - L` |
| `200B` | `STA 2054` | 将 `A` 的结果存储到 `2054` |
| `200E` | `MOV A, D` | `A = D` |
| `200F` | `SBB H` | 借位减：`A = A - H - Carry` |
| `2010` | `STA 2055` | 将 `A` 的结果存储到 `2055` |
| `2013` | `HLT` | 终止程序 |

## 解释
1.  **`LHLD 2050`**: 加载地址为 `2050` 的 `HL` 对。
2.  **`XCHG`**: 用 `DE` 交换 `HL` 对的含量。
3.  **`LHLD 2052`**: 加载地址为 `2052` 的 `HL` 对。
4.  **`MOV A, E`**: 将寄存器 `E` 的内容移到 `A`。
5.  **`SUB L`**: 用寄存器 `L` 的内容减去 `A` 的内容。
6.  **`STA 2054`**: 将累加器的结果存储到内存地址 `2054`。
7.  **`MOV A, D`**: 将寄存器 `D` 的内容移到 `A`。
8.  **`SBB H`**: 用借位寄存器 `H` 的内容减去 `A` 的内容。
9.  **`STA 2055`**: 将累加器的结果存储到内存地址 `2055`。
10. **`HLT`**: 停止执行程序并停止任何进一步的执行。
