# 你对 HTTP 状态码的理解是什么？

> 原文：[https://www.geeksforgeeks.org/what-do-you-understand-by-the-http-status-codes/](https://www.geeksforgeeks.org/what-do-you-understand-by-the-http-status-codes/)

超文本传输协议是应用层的一种协议。它有助于在网络浏览器和网络服务器之间建立通信。当客户端请求任何信息时，浏览器使用数字状态代码发送响应。这些状态代码是从 100 到 599 的 3 位数形式。然后，客户端相应地对响应采取行动。客户可能并不总是完全理解状态代码或响应，但是可以通过看到数字状态代码的最左边/第一个数字来理解响应的性质/类别。

HTTP 状态代码可以用它们的第一个数字分为五个标准类：

*   `1xx`– 这代表一个`信息响应`，即请求已被接收并正在处理，因此我们应该等待最终响应。
*   `2xx`– 这意味着`成功响应`，即请求已被成功接收、理解并接受。
*   `3xx`– 这意味着`重定向响应`，即需要采取一些进一步的操作来完成请求。
*   `4xx`– 这代表`客户端错误`，即请求包含错误的语法或无法实现。
*   `5xx`– 这代表`服务器错误`，即服务器未能满足一个明显有效的请求。

在定义类时，其他两位数字没有如此重要的作用。

**下面提到了每类状态代码的几个例子：**

*   `101`-切换协议- 这意味着服务器已被要求切换协议，并且服务器已同意这样做。
*   `102`-处理中- 这意味着请求可能包含许多子请求，例如文件操作，这些操作可能需要很长时间才能完成。这有助于防止客户端认为请求已超时。
*   `200`–OK– 这是对所有成功 HTTP 请求的标准响应。
*   `202`-已接受- 这意味着请求已被服务器接受但尚未完成。
*   `307`-临时重定向- 这意味着当前发送到服务器的请求应使用另一个 URL 重复。未来的请求仍将发送到原始 URL。
*   `308`-永久重定向– 与前一个不同，当前和所有未来的请求都将被重定向到另一个给定的网站。
*   `404`-未找到- 这仅仅意味着发送的请求未找到，但将来可能会可用。
*   `424`-依赖失败- 这意味着请求失败，因为它依赖于另一个请求，而该请求也失败了。
*   `500`-内部服务器错误- 当发生意外情况且服务器不知道具体是什么时，会出现标准错误消息。
*   `504`-网关超时- 这意味着服务器未及时收到响应。
