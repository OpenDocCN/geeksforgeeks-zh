# 分布式系统中的逻辑时钟

> 原文：[https://www.geeksforgeeks.org/logical-clock-in-distributed-system/](https://www.geeksforgeeks.org/logical-clock-in-distributed-system/)

**逻辑时钟**指的是在分布式系统中的所有机器上实现一个协议，以便机器能够在某个虚拟时间跨度内保持事件的一致顺序。逻辑时钟是一种在分布式系统中捕捉时间和因果关系的机制。分布式系统可能没有物理上同步的全局时钟，因此逻辑时钟允许在这样的系统中对来自不同进程的事件进行全局排序。

## 示例

如果我们计划外出，我们会制定一个完整的行程，规定先去哪个地方，再去哪个地方，依此类推。我们不会先去第二个地点，再返回第一个地点。我们总是遵循预先计划好的程序或组织。同样，我们也应该有组织地在计算机上按顺序执行操作。

假设在一个分布式系统中有10多台计算机，每台计算机都在执行自己的任务。那么，我们如何让它们协同工作呢？一个解决方案就是使用逻辑时钟。

## 事件排序的方法

### 方法一：同步物理时钟

尝试一种方法来同步所有进程的时钟。

这意味着如果一台计算机的时间是下午2:00，那么所有其他计算机的时间也应该是相同的。但这在现实中是完全不可能的，因为无法让所有时钟都保持绝对同步。因此，我们不能依赖这种方法。

### 方法二：为事件分配时间戳

另一种方法是为事件分配时间戳。

考虑这个类比：如果我们给第一个地点编号为1，第二个地点编号为2，第三个地点编号为3，依此类推。那么我们总是知道第一个地点永远是第一，顺序不会改变。同样，如果我们给每台计算机分配一个唯一的编号，那么就可以规定第一台计算机先完成它的过程，然后是第二台，依此类推。

但是，时间戳只有在服从因果关系时才会起作用。

## 什么是因果关系？

因果关系完全基于“发生在前”的关系。

*   当两个事件A和B在单个进程中一前一后发生时，如果`TS(A) < TS(B)`，则表明A发生在B之前。例如，如果A的时间戳是1，那么B的时间戳应该大于1，这符合“发生在前”的关系。
*   考虑两个进程，P1中的事件A和P2中的事件B，条件是`TS(A) < TS(B)`。例如，假设你在下午2:00:00发送一条消息，对方在下午2:00:02接收到。那么显然，`TS(发送者) < TS(接收者)`。

## “发生在前”关系所衍生的属性

*   **传递性**
    如果`TS(A) < TS(B)` 且 `TS(B) < TS(C)`，那么 `TS(A) < TS(C)`。
*   **因果有序关系**
    `a -> b` 表示a发生在b之前。如果a有任何变化，它必定会反映在b中。
*   **并发事件**
    这意味着并非所有事件都按顺序发生，有些事件是同时发生的，即 `A || B`。
