# 作业、任务和流程的区别

> 原文: [https://www.geeksforgeeks.org/difference-between-job-task-and-process/](https://www.geeksforgeeks.org/difference-between-job-task-and-process/)

**前提:** [操作系统](https://www.geeksforgeeks.org/introduction-of-process-management/)中[进程管理](https://www.geeksforgeeks.org/introduction-of-process-management/)与[联动调度](https://www.geeksforgeeks.org/gang-scheduling-in-operating-system/)介绍。

**简介:**
工作是需要完成的任务。
任务是需要完成的工作。
进程是为了特定目的而进行的一系列行动。
工作和任务定义了要完成的内容，而进程定义了完成工作的方式或步骤。

**转向它们在计算中的意义，这里有一个简单的介绍：**

## 1. 进程

*   进程是一个正在执行的程序。程序可以被定义为一组指令。
    程序是被动实体，进程是主动实体。当我们执行一个程序时，它保存在硬盘上；当这个程序进入主存储器时，它就变成了一个进程。
    进程可以出现在硬盘、内存或中央处理器上。
    **示例：**
    在 Windows 中，我们可以在任务管理器中看到每个正在运行的进程。所有在后台运行的进程都可以在任务管理器的“进程”选项卡下看到。
    另一个例子是，当我们在屏幕上执行其他任务时，一个打印机程序在后台运行。那个打印机程序就被称为进程。
*   一个进程在执行时会经历许多状态。其中一些状态是启动、就绪、运行、等待或终止。这些名字没有标准化。这些状态显示在[进程状态转换图或进程生命周期](https://www.geeksforgeeks.org/states-of-a-process-in-operating-systems/)中。
*   可以同时执行多个进程。当多个进程同时执行时，需要决定先执行哪个进程。这被称为进程调度。因此，进程也被称为可调度和可执行单元。
*   一个进程有一定的属性，例如 `process id`、`process state`、`priority` 等。进程也有一个进程内存。进程内存分为 4 个部分——文本部分、数据部分、堆和栈。
*   该进程还便于进程间通信。当执行多个进程时，进程必须使用通信协议进行通信，以保持同步。
*   要深入了解进程的细节，您可以参考：[进程管理介绍](https://www.geeksforgeeks.org/introduction-of-process-management/)。

## 2. 任务

*   任务是正在执行的工作单元。操作系统中的任务可能是进程的同义词。任务是工作的一个子部分。任务组合成一份工作。
*   任务的定义是模糊的，因为它有许多含义。
    任务可能是一个线程、进程、单个作业等等。
    任务在执行时被称为线程。
*   **示例：** 当我们在 Java 中运行一个线程时，它被称为任务。如果打印机打印文档，则称其执行打印任务。当计算机计算用户输入的两个数相加时，也是一项任务（加法任务）。
*   多个任务可以同时一起执行，这就是所谓的多任务处理。当多个任务同时并行执行时，这就是所谓的并行任务。多任务处理也被称为时间共享。多任务是多道程序操作系统的扩展。

## 3. 工作

*   作业是正在执行的完整工作单元。一项工作由许多任务组成，而这些任务又由许多进程组成。作业是批处理模式下的一系列任务。程序是为执行一项工作而编写的。
*   作业的定义也很模糊，因为它也有很多含义。在计算中，作业和任务有时是同义词。
*   **示例：** 计算机的工作是从用户处获取输入，处理数据并提供结果。这项工作可以分为几个小任务：将输入作为一项任务，将数据处理作为另一项任务，将结果输出作为另一项任务。
    这些任务在更小的进程中进一步执行。接受输入的任务涉及许多进程。首先，用户输入信息。然后这些信息被转换成二进制语言。接着，这些信息将被送到中央处理器进行进一步的执行。最后，中央处理器执行必要的操作。因此，作业被分解成任务，这些任务以进程的形式执行。
*   一个作业可以一次执行一个，也可以一次执行多个。单个作业可以称为任务。要同时执行多个作业，需要安排一个作业队列。作业调度程序是一种调度作业的应用程序。作业调度也称为批处理调度。

工作、进程和任务的概念是相互关联的。工作、任务和进程可以根据上下文被认为是相同或不同的。进程是操作系统的一个独立实体。如果一个任务是独立的，它可以被称为一个进程。如果要执行的工作是单个工作单元，则该作业可以称为任务。一个进程或一组进程可以称为任务，一组任务可以称为作业。
