# 可行前缀

> 原文: [https://www.geeksforgeeks.org/viable-prefix-in-bottom-up-parsing/](https://www.geeksforgeeks.org/viable-prefix-in-bottom-up-parsing/)

## 可行前缀的定义

**可行前缀**是右句形式的前缀，它不会延续到该句形式最右边句柄的右端。

这显然意味着一个可行的前缀在其最右端有一个句柄。并非所有正确句子形式的前缀都能出现在`移位-缩减解析器`的堆栈上。

这可以通过下面的例子来说明。

## 示例一

```
A=>B+id=>(E)+id  
(最右推导，句型) 
```

`自底向上解析器`给出了反向最右推导，因此在上面的例子中，如果我们在反向最右推导过程中得到字符串`(E)+id`，那么执行以下操作:

| 已执行的操作 | 堆栈 | 评论 |
| --- | --- | --- |
| `(.E)+id` | `(` | 移位`(` |
| `(E).+id` | `(E)` | `E` 入栈 |
| `(E).+id` | `(E)` | 移位`)` |
| `B.+id` | `B` | 将`(E)`归约为`B` |
| `B+.id` | `B +` | 移位`+` |
| `B+id.` | `B + id` | 移位`id` |
| `A` | `A` | 将`B + id`归约为`A` |

正如我们在上表中看到的，在将`+`移动到堆栈之前，我们已经将`(E)`归约到了`B`。因此，我们在堆栈上只能有`(`，`(E)`，`(B)`，但在堆栈上不能有`(E)+`，因为`(E)`是一个句柄，堆栈中的项目不能超过该句柄。所以这里`(`、`(E)`、`(B)`都是句柄`(E)`的可行前缀，只有这些前缀出现在`移位-缩减解析器`的堆栈中。

因此，我们不断移动项目，直到我们到达句柄或出现错误。一旦到达一个句柄，我们就使用合适的产生式用非终结符归约它。因此，可行的前缀有助于采取适当的归约决策。只要堆栈包含这些前缀，就不会有任何错误。

`LR(0)`自动机可以识别所有可行的前缀。因此，给定`LR(1)`解析器的可行前缀集是一种常规语言。栈与有限状态机的这种结合实际上是一种下推自动机，它实际上用于识别一种上下文无关的语言。

## 示例二

以下示例说明了给定语法的所有可行前缀。

给定语法:

```
S -> AA
A -> bA | a 
```

给定字符串:

```
bbbaa 
```

**解决方案:**
正如我们所知，自底向上解析是给定语法的最右推导的逆过程。因此，我们使用字符串最右推导的逆过程来演示这个示例。

| 序号 | 带句柄的反向最右推导 | 可行前缀 | 评论 |
| --- | --- | --- | --- |
| 1. | `S -> bbb **a** a` | `b`、`bb`、`bbb`、`bbba` | 这里，`a`是句柄，因此可行前缀不能超过`a`。 |
| 2. | `S -> bb **bA** a` | `b`、`bb`、`bbb`、`bbbA` | 这里，`bA`是句柄，因此可行前缀不能超过`bA`。 |
| 3. | `S -> b **bA** a` | `b`、`bb`、`bbA` | 这里同样，`bA`是句柄，因此可行前缀不能超过`bA`。 |
| 4. | `S -> **bA** a` | `b`，`bA` | 这里同样，`bA`是句柄，因此可行前缀不能超过`bA`。 |
| 5. | `S -> A **a**` | `a`，`Aa` | 这里，`a`是句柄，因此可行前缀不能超过`a`。 |
| 6. | `S -> **AA**` | `A`，`AA` | 这里，`AA`是句柄，因此可行前缀不能超过`AA`。 |
