# 数据库管理系统中 OLAP 系统的类型

> 原文：[https://www.geeksforgeeks.org/types-of-olap-systems-in-dbms/](https://www.geeksforgeeks.org/types-of-olap-systems-in-dbms/)

OLAP 服务器主要有以下三种类型：

## 1. 关系型 OLAP (ROLAP) – 基于星型模式

`ROLAP` 基于这样一个前提：数据无需以多维方式存储即可进行多维查看，并且可以利用成熟的关系数据库技术来处理数据的多维性。在 `ROLAP` 中，数据存储在关系数据库中。本质上，每一次切片和切块操作都等价于在 `SQL` 语句中添加一个 `WHERE` 子句。`ROLAP` 能够处理大量数据，并可以利用关系数据库固有的功能。

## 2. 多维 OLAP (MOLAP) – 基于立方体

`MOLAP` 将数据存储在磁盘上的专用多维数组结构中。`OLAP` 操作依赖于数组的随机访问能力。数组元素由维度实例确定，与每个单元格关联的事实数据或度量值通常存储在相应的数组元素中。在 `MOLAP` 中，多维数组通常根据轴的嵌套遍历顺序以线性分配方式存储。

但是与 `ROLAP` 不同，`ROLAP` 只存储具有非零事实的记录，而所有数组元素都在 `MOLAP` 中定义。因此，数组通常倾向于稀疏，空元素占据了大部分空间。由于存储和检索成本在评估在线性能效率时都很重要，`MOLAP` 系统通常包括高级索引和哈希等配置，以便在执行查询时定位数据以处理稀疏数组。`MOLAP` 立方体支持快速的数据检索，最适合切片和切块操作，并且可以执行复杂的计算。所有计算都是在创建多维数据集时预先生成的。

## 3. 混合型 OLAP (HOLAP)

`HOLAP` 是 `ROLAP` 和 `MOLAP` 的结合。`HOLAP` 服务器允许存储大量详细数据。一方面，`HOLAP` 利用了 `ROLAP` 更强的可扩展性。另一方面，`HOLAP` 利用了立方体技术以获得更快的性能和汇总型信息。由于详细数据保存在关系数据库中，因此 `HOLAP` 的立方体比 `MOLAP` 的更小。数据库以尽可能功能化的方式存储数据。

## 一些其他类型的 OLAP

### Web OLAP (WOLAP)

它是一种基于 `Web` 浏览器的技术。传统的 `OLAP` 应用程序通过客户端/服务器访问，而这种 `OLAP` 应用程序通过 `Web` 浏览器访问。它是一个三层架构，包括客户端、中间件和数据库服务器。这种 `OLAP` 方式最吸引人的特点是（过去时态，因为现在很少有产品这样分类）客户端投资相当低（“只需要一个浏览器”）以及增强的数据连接可访问性。基于 `Web` 的应用程序不需要在客户端机器上部署，只需要一个 `Web` 浏览器和到内联网或互联网的网络连接。

### Desktop OLAP (DOLAP)

`DOLAP` 代表桌面分析处理。用户可以从源下载数据并在其桌面上处理数据集。与其他 `OLAP` 应用程序相比，其功能有限，成本也更低。

### Mobile OLAP (MOLAP)

`MOLAP` 提供无线功能或支持移动设备。用户可以通过移动设备工作和访问数据。

### 空间 OLAP (SOLAP)

将地理信息系统 (`GIS`) 和 `OLAP` 的功能合并到单一用户界面 `SOLAP` 中。创建 `SOLAP` 是因为数据以字母数字、图像和向量的形式出现。这提供了对驻留在空间数据库上的数据的简单而快速的探索。
