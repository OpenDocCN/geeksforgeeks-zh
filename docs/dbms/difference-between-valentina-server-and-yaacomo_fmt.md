# Valentina 服务器和 Yaacomo 的区别

> 原文: [https://www.geeksforgeeks.org/difference-between-valentina-server-and-yaacomo/](https://www.geeksforgeeks.org/difference-between-valentina-server-and-yaacomo/)

## 1. `Valentina Server`
它是对象关系数据库和报表服务器以及快速对象关系数据库服务器。它包括集成数据库服务器，`Valentina`数据库服务器和`SQLite`数据库服务器。它是功能齐全、符合标准的基于多用户访问`SQLite`的数据库服务器。

## 2. `Yaacomo`
它是一个基于关系、`OpenCL`的内存数据库管理系统，用于实时分析和大数据解决方案，是作为即插即用解决方案开发的。`Yaacomo`旨在通过并行计算有效利用硬件。

## `Valentina Server` 与 `Yaacomo` 的区别

| 编号 | `Valentina Server` | `Yaacomo` |
| :--- | :--- | :--- |
| 1 | 由`Paradigma`软件开发，最初于`1994`年发布，当前版本的`Valentina Server`是`5.7.5`。 | 由`Q2WEB GmbH`开发，最初于`2009`年发布。 |
| 2 | `Valentina Server`的服务器操作系统是`Linux`、`OS X`、`Windows`。 | `Yaacomo`的服务器操作系统是`Android`、`Linux`、`Windows`。 |
| 3 | 它的主要数据库模型是关系数据库管理系统。 | 它的主要数据库模型是关系数据库管理系统。 |
| 4 | `Valentina Server`支持`.Net`、`C`、`C#`、`C++`、`Objective-C`、`PHP`、`Ruby`、`Visual Basic`、`Visual Basic.NET`编程语言。 | 它不支持任何编程语言。 |
| 5 | `Valentina Server`的`API`等访问方式是`ODBC`。 | `Yaacomo`的`API`和其他访问方法是`JDBC`、`ODBC`。 |
| 6 | 是的，`Valentina Server`有一个数据模式。 | `Yaacomo`有一个数据模式。 |
| 7 | 它没有实现语言——`C#`。 | 它没有任何实现语言。 |
| 8 | `Valentina Server`中缺少事务概念。 | 它有事务概念——是`ACID`（原子性、一致性、隔离性和持久性）。 |
| 9 | `Valentina Server`没有一致性概念。 | `Yaacomo`有立即一致性的概念。 |
| 10 | 是的，它有外键。 | 它持有外键。 |
| 11 | `Valentina Server`中缺少复制方法。 | `Yaacomo`的复制方法是主从复制。 |
| 12 | `Valentina Server`中缺少分区方法。 | 水平分割是`Yaacomo`的一种分割方法。 |
