# 数据库不可用导致的问题

> 原文：[https://www.geeksforgeeks.org/problems-due-to-database-unavailability/](https://www.geeksforgeeks.org/problems-due-to-database-unavailability/)

数据库安全以保护数据库免受意外、偶然或有意威胁而闻名。这些威胁可能基于也可能不基于计算机。威胁是将影响数据库安全和组织运作的任何情况或事件。这种威胁可能会给组织造成巨大损失。数据库不可用将严重影响组织的绩效。

不可用会给组织带来巨大损失。

## Data security risks
`Database security`是整个组织数据库的主要部分。组织必须从数据库的角度识别所有可能导致未来风险的因素以及数据库的所有薄弱元素。

## Data tampering
`Data tempering`是故意修改数据的行为。当数据分布式存储时，数据篡改的可能性更大，因为数据会从一个站点移动到另一个站点。因此，隐私对于保护数据至关重要，以防止非目标用户查看或修改数据。

## Data theft
数据必须安全地传输和存储，以便一些重要信息（如银行详细信息）不会被窃取。

## False user identity
在分布式系统中，非目标用户可以轻易地通过使用虚假身份获取敏感和重要信息。这些非目标用户可以获取重要信息，他们可以窃取银行详细信息、可在其他地方使用的唯一代码等。

## Password related threats
在大型组织中，用户出于安全考虑需要为他们使用不同服务记住多个密码。用户可以通过以下方式管理多个密码：
*   用户可以使用容易猜到的密码。
*   用户可以为每个站点选择相同的密码，也可以使用标准密码。

然而，多重密码昂贵、难以管理且复杂。

## Unauthorized Access to Tables and columns
数据库可能包含一些对未授权用户受限的数据。这些数据可能是机密表、机密行或列。应该存在保护机密数据的可能性。

## Lack of accountability
如果系统管理员无法找到用户活动，那么用户就不能对其活动负责。应该有一种方法来监控用户执行的所有操作。

## 复杂的用户管理需求
系统必须能够支持大量用户，因此必须具有可扩展性。在大规模系统环境中，管理用户帐户和密码的负担使得系统容易受到攻击和错误的影响。
