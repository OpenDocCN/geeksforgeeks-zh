# 数据挖掘中数据源的类型

> 原文: [https://www.geeksforgeeks.org/types-of-sources-of-data-in-data-mining/](https://www.geeksforgeeks.org/types-of-sources-of-data-in-data-mining/)

在这篇文章中，我们将讨论在数据挖掘过程中使用的不同数据源。来自多个来源的数据被集成到一个名为`数据仓库`的公共来源中。

**我们来讨论一下可以挖掘什么类型的数据:**

*   `Flat file`
*   `关系数据库`
*   `数据仓库`
*   `事务数据库`
*   `多媒体数据库`
*   `空间数据库`
*   `时间序列数据库`
*   `World Wide Web(WWW)`

## Flat file

*   `Flat file`被定义为文本或二进制形式的数据文件，其结构可以被数据挖掘算法轻松提取。
*   存储在`flat file`中的数据之间没有关系或路径，就像如果关系数据库存储在`flat file`中，表之间将没有关系。
*   `Flat file`由数据字典表示。示例：`CSV file`。
*   **应用**：用于数据仓库中的数据存储和服务器之间的数据传输。

## 关系数据库

*   `关系数据库`被定义为以表和行列形式组织的数据集。
*   `关系数据库`中的物理模式是定义表结构的模式。
*   `关系数据库`中的逻辑模式是定义表之间关系的模式。
*   `关系数据库`的标准`API`是`SQL`。
*   **应用**：数据挖掘、`ROLAP`模型等。

## 数据仓库

*   `数据仓库`被定义为从多个来源集成的数据集合，用于查询和决策。
*   `数据仓库`有三种类型：企业数据仓库、数据集市和虚拟数据仓库。
*   更新`数据仓库`中的数据有两种方式：查询驱动和更新驱动。
*   **应用**：业务决策、数据挖掘等。

## 事务数据库

*   `事务数据库`是一个按时间戳、日期等组织的数据集合，用于表示数据库中的事务。
*   当事务未完成或未提交时，这种类型的数据库可以回滚或撤销其操作。
*   高度灵活的系统，用户可以在不更改任何敏感信息的情况下修改信息。
*   遵守数据库管理系统`ACID`特性。
*   **应用**：银行、分布式系统、对象数据库等。

## 多媒体数据库

*   `多媒体数据库`由音频、视频、图像和文本媒体组成。
*   它们可以存储在面向对象的数据库中。
*   它们用于以预定义的格式存储复杂信息。
*   **应用**：数字图书馆、视频点播、新闻点播、音乐数据库等。

## 空间数据库

*   存储地理信息。
*   以坐标、拓扑、线、多边形等形式存储数据。
*   **应用**：地图、全球定位等。

## 时间序列数据库

*   `时间序列数据库`包含股票交易数据和用户记录的活动。
*   处理由时间、日期等索引的数字数组。
*   需要实时分析。
*   **应用领域**：`ExtremeDB`、`Graphite`、`InfluxDB`等。

## WWW

*   `WWW`指的是文档和资源的集合，例如音频、视频、文本等，这些资源通过`URL`通过`Web`浏览器标识，由`HTML`页面链接并通过互联网网络访问。
*   它是最异构的存储库，因为它从多个资源收集数据。
*   随着数据量的增加和变化，它本质上是动态的。
*   **应用**：在线购物、求职、研究、学习等。
