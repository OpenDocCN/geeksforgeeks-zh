# Valentina 服务器和 VelocityDB 的区别

> 原文：[https://www.geeksforgeeks.org/difference-between-valentina-server-and-velocitydb/](https://www.geeksforgeeks.org/difference-between-valentina-server-and-velocitydb/)

## 1. Valentina 服务器
它是一个对象关系数据库和报表服务器以及快速对象关系数据库服务器。它包括集成数据库服务器，`Valentina`数据库服务器和`SQLite`数据库服务器。它是一个功能齐全、符合标准的基于`SQLite`的多用户访问数据库服务器。

## 2. VelocityDB
它是一个`.NET`对象数据库，可以嵌入/分发并扩展到图形数据模型(`VelocityGraph`)数据库速度快，内存消耗低。`VelocityDB`是一个性能极高、易于使用、可扩展、可嵌入和可分发的对象数据库系统。

## Valentina Server 与 VelocityDB 的区别

| 编号 | Valentina Server | VelocityDB |
| :--- | :--- | :--- |
| 1 | 由`Paradigma`软件开发，最初于`1994`年发布。 | 由`VelocityDB Inc.`开发，最初于`2011`年发布。 |
| 2 | `Valentina Server`的当前版本是`5.7.5`。 | `VelocityDB`的当前版本是`7.x` |
| 3 | `Valentina Server`的服务器操作系统是`Linux`、`OS X`、`Windows`。 | `VelocityDB`的服务器操作系统是任何支持`.Net`的系统。 |
| 4 | 它的主要数据库模型是关系数据库管理系统。 | 它的主要数据库模型是图形数据库和面向对象数据库。 |
| 5 | `Valentina Server`支持`.Net`、`C`、`C#`、`C++`、`Objective-C`、`PHP`、`Ruby`、`Visual Basic`、`Visual Basic.NET`编程语言。 | `VelocityDB`支持`.Net`编程语言。 |
| 6 | `Valentina Server`的`API`和其他访问方式是`ODBC`。 | `VelocityDB`的`API`和其他访问方法是`.Net`。 |
| 7 | 是的，`Valentina Server`有一个数据模式。 | 是的，`VelocityDB`有一个数据模式。 |
| 8 | 它没有实现语言——`C#`。 | 它有实现语言——`C#`。 |
| 9 | `Valentina Server`中缺少事务概念。 | 事务概念是`ACID`（原子性、一致性、隔离性和持久性）。 |
| 10 | `Valentina Server`没有一致性概念。 | `VelocityDB`具有即时一致性概念。 |
| 11 | 是的，它有外键。 | 不，它没有外键。 |
| 12 | `Valentina Server`中缺少复制方法。 | 缺少复制方法。 |
| 13 | `Valentina Server`中缺少分区方法。 | 分片是`VelocityDB`的一种划分方法。 |
