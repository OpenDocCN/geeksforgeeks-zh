# 数据库管理系统中的模式设计策略

> 原文: [https://www.geeksforgeeks.org/strategies-for-schema-design-in-dbms/](https://www.geeksforgeeks.org/strategies-for-schema-design-in-dbms/)

设计模式时要考虑各种策略。这些策略大多遵循一种增量方法，也就是说，它们必须从需求派生的一些模式结构开始，然后在它们的基础上进行增量修改、细化或构建。让我们讨论其中的一些策略：

## 1. 自顶向下策略

在此策略中，我们基本上从一个包含高度抽象的模式开始，然后应用连续的自顶向下细化。让我们通过一个例子来理解这一点：我们可能只指定少数几个层级的实体类型，然后指定它们的属性，将其拆分为更低层级的实体类型和关系。将实体类型细化为子类的`specialization`过程也是此策略的一个例子。

## 2. 自底向上策略

在此类策略中，我们基本上从基本的抽象开始，然后不断向此抽象添加内容。例如，我们可能从属性开始，并将这些属性分组为实体类型和关系。随着设计的推进，我们还可以在实体类型之间添加新的关系。基本的例子是将实体类型`generalizing`为更高层级的泛化超类的过程。

## 3. 由内向外策略

这是自底向上策略的一种特殊情况，其注意力主要集中在最明显的核心概念集合上。建模然后基本上通过考虑现有概念附近的新概念向外扩展。我们可以在模式中指定几个明显易见的实体类型，并通过添加与之相关的其他实体类型和关系来继续。

## 4. 混合策略

在整个设计过程中不使用任何特定策略，而是根据自顶向下策略对需求进行划分，然后为每个划分根据自底向上策略设计部分模式，最后将各种模式组合起来。
