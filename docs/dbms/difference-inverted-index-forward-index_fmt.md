# 倒推指数与顺推指数的差异

> 原文：`https://www.geeksforgeeks.org/difference-inverted-index-forward-index/`

## [【倒排索引】](https://www.geeksforgeeks.org/inverted-index/)

1.  它是一种数据结构，存储从词语到文档或文档集合的映射，即引导你从词语找到文档。
2.  构建倒排索引的步骤如下：
    *   获取文档并收集所有词语。
    *   检查每个词语，如果存在，则向索引添加文档引用，否则在索引中为该词语创建新条目。
    *   对所有文档重复上述步骤并对词语进行排序。
3.  索引过程较慢，因为它首先检查词语是否存在。
4.  搜索非常快。
5.  倒排索引示例：

```
    Word                              Documents
    hello                             doc1      
    sky                               doc1, doc3
    coffee                            doc2
    hi                                doc2
    greetings                         doc3
```

它在索引中不存储重复的关键字。

6.  倒排索引的现实例子：
    *   书后的索引。
    *   反向查找。

## 正向索引

1.  它是一种数据结构，存储从文档到词语的映射，即引导你从文档找到词语。
2.  建立正向索引的步骤是：
    *   获取文档并收集所有关键词。
    *   将所有关键词附加到该文档的索引条目中。
    *   对所有文档重复上述步骤。
3.  索引过程相当快，因为它在向前推进时只附加关键词。
4.  搜索相当困难，因为它必须查看索引的每个内容以检索与词语相关的所有页面。
5.  正向索引示例：

```
    Document                          Keywords
    doc1                              hello, sky, morning      
    doc2                              tea, coffee, hi
    doc3                              greetings, sky
```

它在索引中存储重复的关键字。单词“sky”被存储多次。

6.  正向索引的现实例子：
    *   书中的目录。
    *   DNS查找。

## 正向索引和反向索引的相似度

*   两者都用于在文档或文档集合中搜索文本。
