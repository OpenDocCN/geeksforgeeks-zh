# SQL 和 NoSQL 的区别

> 原文: [https://www.geeksforgeeks.org/difference-between-sql-and-nosql/](https://www.geeksforgeeks.org/difference-between-sql-and-nosql/)

先决条件 – [`SQL`](https://www.geeksforgeeks.org/sql-tutorial/)、[`NoSQL`](https://www.geeksforgeeks.org/introduction-to-nosql/)

在选择数据库时，最大的决策是选择`关系型(SQL)`或非关系型(`NoSQL`)数据结构。虽然这两个数据库都是可行的选择，但用户在做出决定时必须记住两者之间的某些关键差异。

## 主要区别:

### 1. Type –
`SQL`数据库主要被称为关系数据库(`RDBMS`)；而`NoSQL`数据库主要被称为非关系或分布式数据库。

### 2. Language –
`SQL`数据库基于结构化查询语言(`SQL`)来定义和操作数据。从一方面看，这种语言极其强大。`SQL`是最通用和广泛使用的选项之一，这使其成为复杂查询的安全选择。但从另一方面看，它可能是限制性的。`SQL`要求您使用预定义的模式来确定数据的结构，然后才能使用它。此外，您的所有数据都必须遵循相同的结构。这可能需要大量的前期准备，这意味着结构的改变对整个系统来说将是困难且破坏性的。

`NoSQL`数据库具有非结构化数据的动态模式。数据以多种方式存储，这意味着它可以是面向文档的、面向列的、基于图形的，或者组织为键值存储。这种灵活性意味着无需先定义结构就可以创建文档。每个文档也可以有自己独特的结构。语法因数据库而异，您可以随时添加字段。

### 3. The Scalability –
在几乎所有情况下，`SQL`数据库都是纵向可扩展的。这意味着您可以通过增加`RAM`、`CPU`或`SSD`来增加单台服务器的负载。另一方面，`NoSQL`数据库是水平可扩展的。这意味着您可以通过分片或在`NoSQL`数据库中添加更多服务器来处理更多流量。这类似于在同一栋建筑中增加更多楼层， versus 在社区中增加更多建筑。因此，`NoSQL`最终可以变得更大、更强大，使其成为大型或不断变化的数据集的首选数据库。

### 4. The Structure –
`SQL`数据库是基于表的，而`NoSQL`数据库是键值对、基于文档、图数据库或宽列存储。这使得关系型`SQL`数据库成为需要多行事务的应用程序（如会计系统）或为关系结构构建的遗留系统的更好选择。

### 5. Property followed –
`SQL`数据库遵循`ACID`属性（原子性、一致性、隔离性和持久性），而`NoSQL`数据库遵循`Brewers CAP`定理（一致性、可用性和分区容差）。

### 6. Support –
所有`SQL`数据库都能从其供应商那里获得强大的支持。此外，还有许多独立顾问可以帮助您进行大规模部署的`SQL`数据库，但对于某些`NoSQL`数据库，您仍然必须依赖社区支持，并且只有有限的外部专家可用于设置和部署大规模的`NoSQL`部署。

`SQL`数据库的一些例子包括`PostgreSQL`、`MySQL`、`Oracle`和微软的`SQL Server`。`NoSQL`数据库的例子包括`Redis`、`RavenDB`、`Cassandra`、`MongoDB`、`BigTable`、`HBase`、`Neo4j`和`CouchDB`。

## SQL vs NoSQL 的关键亮点:

| 结构化查询语言 | NoSQL |
| --- | --- |
| 关系数据库管理系统 | 非关系或分布式数据库系统。 |
| 这些数据库有固定的或静态的或预定义的模式 | 他们有动态模式 |
| 这些数据库不适合分层数据存储。 | 这些数据库最适合分层数据存储。 |
| 这些数据库最适合复杂的查询 | 这些数据库不太适合复杂的查询 |
| 纵向可扩展 | 水平可伸缩 |
| 遵循 `ACID` 属性 | 遵循 `CAP`(一致性、可用性、分区容差) |
