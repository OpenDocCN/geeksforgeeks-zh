# 如何从数据库中存储和提取 XML 文档

> 原文: [https://www.geeksforgeeks.org/how-to-store-and-extract-xml-documents-from-databases/](https://www.geeksforgeeks.org/how-to-store-and-extract-xml-documents-from-databases/)

我们可以使用不同的方法来组织 `XML` 文档的内容，以便于后续的查询和检索。

以下是从数据库中存储和提取 `XML` 文档的方法。

1.  在第一种方法中，我们可以使用数据库管理系统，其中我们将文档作为文本来存储，并使用关系型和对象型 `DBMS` 将整个 `XML` 文档作为 `DBMS` 记录和对象中的文本字段存储。在此过程中，我们可以使用文档处理作为数据库管理系统的特殊模块，它存储无模式且以文档为中心的 `XML` 文档。
2.  在第二种方法中，我们再次使用数据库管理系统，但这次我们可以用于将文档内容存储为数据元素。此过程遵循 `XML` 或 `XML DTD` 模式，并且所有文档都具有相同的结构，我们可以设计一个关系对象数据库来存储 `XML` 文档中的叶级数据元素。为此，我们需要一个映射算法来设计一个与 `XML` 文档结构兼容的数据库模式。

它通常用于从为 `XML` 或 `DTD` 模式指定的存储数据中重新创建 `XML` 文档。这有助于实现不属于数据库管理系统的内部数据库管理系统模块或中间件。

3.  在第三种方法中，我们可以使用不同类型的数据库管理系统，该系统基于为设计和实现以及设计用于存储原生 `XML` 数据的主系统而基于分层（树）模型。该系统也称为原生 `XML DBMS`，在此过程中，系统可能涉及索引和查询技术，这些技术适用于所有类型的 `XML` 文档。

它可能包括数据压缩技术，该技术被高度用于减小存储文档的大小。有各种软件提供原生的可扩展标记语言存储选项，如动态应用平台的 `Tamino` 软件公司或 `Excelon` 提供原生的可扩展标记语言数据库管理系统功能，`oracle` 也提供原生的可扩展标记语言存储选项。

4.  在第四种方法中，我们可以从预先存在的关系数据库创建和定制 [`XML` 文档](https://www.geeksforgeeks.org/xml-basics/) 并进行发布，因为关系数据库已经存储了大量数据，这些数据是其中的一部分，我们需要将格式化数据作为文档显示在 `Web` 上或进行交换。

这个过程用于划分中间件软件层，以调整一个 `XML` 文档和关系数据库之间的转换。
