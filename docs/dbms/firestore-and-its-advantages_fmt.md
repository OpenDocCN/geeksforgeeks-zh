## 简介:

[`Firestore`](https://www.geeksforgeeks.org/realtime-database-vs-firestore/) 是一个由谷歌开发的 [`NoSQL`](https://www.geeksforgeeks.org/introduction-to-nosql/) [`数据库`](https://www.geeksforgeeks.org/dbms/)，作为 firebase 数据库的替代品，firebase 数据库已经获得了巨大的人气。它旨在提供更好的开发人员体验并简化开发过程。它是存储数据的强大工具。

## 优势:

现在，我们来讨论一下 Firestore 的优势。

### 异步查询

*   Firestore 的一个基本功能是支持从移动设备和互联网进行实时数据检索。
*   执行查询时，你可以为其附加一个监听器，该监听器接收实时回调，因此每次查询监听器发生变化时你都会收到通知。
*   因此，可以从文档中实现实时更新。

### 提供出色的数据处理能力

*   Cloud Firestore 使得处理和组织数据更加容易。
*   Firestore 将数据存储在组织成集合的文档中，并提供了一个高性能的查询引擎，开发者可以用它对 NoSQL 数据库运行复杂的查询。
*   Cloud Firestore 针对存储大量小型文档集合进行了优化，在数据结构方面提供了极大的灵活性。

### 离线支持

*   Firestore 提供离线同步功能，帮助用户在没有网络连接的情况下与应用程序交互。
*   因此，用户可以离线存储数据，并在连接后立即将其同步到数据库。
*   Firestore 库处理所有同步、数据合并、通知等问题，使应用程序在未连接到互联网时也能完美运行。

### 专为扩展而设计

*   Firestore 是一个快速的 NoSQL 数据库，设计为可与 Google Cloud Platform 一起扩展，根据负载提供自动的水平扩展。
*   它提供了全球可扩展性和强大的可靠性。

然而，市场上有各种各样的数据库，你无法知道哪一个是最好的。就像编程中的一切一样——你需要权衡利弊，考虑未来，选择最适合你的应用程序的东西。你必须了解你对项目的需求。
