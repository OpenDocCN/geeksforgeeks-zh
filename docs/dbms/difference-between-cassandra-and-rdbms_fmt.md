# 卡珊德拉和 RDBMS 的区别

> 原文：[https://www.geeksforgeeks.org/difference-between-cassandra-and-rdbms/](https://www.geeksforgeeks.org/difference-between-cassandra-and-rdbms/)

## 1. `Cassandra`

`Cassandra` 是一个高性能、高可扩展性的分布式 `NoSQL` 数据库管理系统。`Cassandra` 处理非结构化数据，它处理大量的传入数据。在 `Cassandra` 中，写入许多位置的数据也来自许多位置，在此行中代表复制单位，在此列中代表存储单位。

## 2. `关系数据库管理系统`

关系数据库管理系统（`RDBMS`）是为关系数据库设计的数据库管理系统或软件，使用结构化查询语言（`SQL`）对数据库进行查询和维护。它处理结构化数据，并处理中等的传入数据速度。在关系数据库管理系统中，数据主要写在一个位置，数据也来自一个/几个位置，行代表一个记录，列代表一个属性。

## 卡珊德拉和 RDBMS 的区别

| 编号 | `Cassandra` | `关系型数据库管理系统` |
| :--- | :--- | :--- |
| 01. | `Cassandra` 是一个高性能和高度可扩展的分布式 `NoSQL` 数据库管理系统。 | 关系数据库管理系统是为关系数据库设计的数据库管理系统或软件。 |
| 02. | `Cassandra` 是一个 `NoSQL` 数据库。 | 关系数据库管理系统使用 `SQL` 来查询和维护数据库。 |
| 03. | 它处理非结构化数据。 | 它处理结构化数据。 |
| 04. | 它具有灵活的模式。 | 它有固定的模式。 |
| 05. | `Cassandra` 拥有主从核心架构，意味着没有单点故障。 | `RDBMS` 具有主从核心架构意味着单点故障。 |
| 06. | `Cassandra` 处理大量传入数据的速度。 | `RDBMS` 处理中等的传入数据速度。 |
| 07. | 在 `Cassandra` 中，有限的数据源意味着数据来自许多地方。 | 在关系数据库管理系统中，有各种数据源，意味着数据来自一个/几个位置。 |
| 08. | 它支持简单的事务。 | 它支持复杂的嵌套事务。 |
| 09. | 在 `Cassandra`，最外层的容器是键空间。 | 在关系数据库管理系统中，最外层的容器是数据库。 |
| 10. | `Cassandra` 遵循分散部署。 | 关系数据库管理系统遵循集中式部署。 |
| 11. | 在 `Cassandra` 数据写在很多地方。 | 在关系数据库管理系统中，数据主要写在一个位置。 |
| 12. | 在 `Cassandra` 行代表一个复制单位。 | 在关系数据库管理系统中，一行代表一条记录。 |
| 13. | 在 `Cassandra`，列代表一个存储单位。 | 在 `RDBMS` 中，列代表一个属性。 |
| 14. | 在 `Cassandra` 中，关系是用集合来表示的。 | 在关系数据库管理系统中，关系用键和连接等来表示。 |
