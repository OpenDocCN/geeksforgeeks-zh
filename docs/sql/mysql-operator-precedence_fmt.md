# MySQL 运算符优先级

> 原文: [https://www.geeksforgeeks.org/mysql-operator-precedence/](https://www.geeksforgeeks.org/mysql-operator-precedence/)

运算符优先级指定当表达式中两个或多个具有不同优先级的运算符相邻时，运算符的求值顺序。

例如，与`(1+2)/3`相比，`1+2/3`给出了不同的结果。就像所有其他编程语言 C，C++，Java 等一样，MySQL 也有一个优先规则。

## 运算符优先级表

下表描述了 MySQL 中从最高到最低的运算符优先级。同一组中的运算符具有相同的优先级。

| **操作符** | **描述** |
| --- | --- |
| `INTERVAL` | 返回小于第一个参数的参数索引 |
| `BINARY`<br>`COLLATE` | 这是一种存储二进制字节字符串的类型<br>无论默认的比较排序规则是什么，该子句都会覆盖 |
| `!` | 负值 |
| `-`<br>`~` | 它改变操作数的符号<br>它反转操作数的位 |
| `^` | 按位异或 |
| `*`<br>`/`<br>`DIV`<br>`%`, `MOD` | 乘法运算符<br>除法运算符<br>整数除法(丢弃除法的小数部分)<br>模运算符 |
| `-`<br>`+` | 减运算符<br>加法运算符 |
| `<<`<br>`>>` | 向左移动(BIGINT)数字或二进制字符串<br>向右移动(BIGINT)数字或二进制字符串 |
| `&` | 按位“与” |
| `\|` | 按位“或” |
| `=`<br>`<=>`<br>`>=`<br>`>`<br>`<=`<br>`<`<br>`<>`, `!=`<br>`IS`<br>`LIKE`<br>`REGEXP`<br>`IN` | 比较运算符<br>空-安全等于运算符<br>大于/大于或等于<br>小于/小于或等于<br>不等于运算符<br>根据布尔值<br>测试值模式匹配运算符<br>将字符串表达式与正则表达式进行匹配<br>检查列表中是否存在值 |
| `BETWEEN`<br>`CASE`, `THEN`, `ELSE` | 检查数值是否在数值范围内<br>Case 运算符 |
| `NOT` | 否定价值 |
| `AND`, `&&` | 逻辑“与” |
| `XOR` | 逻辑异或 |
| `OR`, `\|\|` | 逻辑或 |
| `:=`<br>`=` | 赋值(作为更新语句中 SET 语句/SET 子句的一部分)<br>赋值 |

这些运算符优先规则极大地影响了我们的 MySQL 查询。如果不知道运算符优先级，我们可能会得到意想不到的结果。

## 示例：学生表查询

要理解这一点，请考虑下表`student`。

| **id** | **name** | **marks** |
| --- | --- | --- |
| 1 | Pappal | 12 |
| 2 | Kash | 9 |
| 3 | Reeta | 19 |
| 4 | Piyush | 15 |
| 5 | Shanu | 5 |
| 6 | Punit | 7 |

从上表中，我们需要那些分数大于 10 分，名字以“p”或“s”开头的学生的成绩。所以，它的查询可以写成：

```sql
SELECT * 
FROM student 
WHERE marks>10 AND name LIKE 'p%' 
                   OR name LIKE 's%';
```

**结果：** 会产生想要的结果：

| **id** | **name** | **marks** |
| --- | --- | --- |
| 1 | Pappal | 12 |
| 4 | Piyush | 15 |
| 6 | Punit | 7 |

该结果集与查询中预期的不同。因为它给出了学生分数低于 10 分的“Shanu”结果，这是不必要的。由于运算符`AND`的优先级高于`OR`，上述查询给出了所有分数大于 10 且姓名以“s”开头的学生的结果，此外，输出中还给出了姓名以“p”开头的学生的结果。

所以，圆括号的作用来了。上述查询可以写成：

```sql
SELECT * 
FROM student 
WHERE marks>10 AND (name LIKE 'p%' 
                       OR name LIKE 's%');
```

**结果：** 会产生想要的结果：

| **id** | **name** | **marks** |
| --- | --- | --- |
| 1 | Pappal | 12 |
| 4 | Piyush | 15 |

因此，可以通过使用圆括号来覆盖运算符的这种优先级。
