# Preact 和 Angular 的差异

> 原文: [https://www.geeksforgeeks.org/difference-between-preact-and-angular/](https://www.geeksforgeeks.org/difference-between-preact-and-angular/)

`Angular` 是谷歌开发的前端框架。它确实有助于应用程序的快速开发，并且基于 `MVC` 框架。`Angular` 完全是用 `Typescript` 编写的。

**功能:**

*   它是 `AngularJS`。
*   它具有双向绑定，依赖注入和数据绑定减少了需要编写的代码量。
*   `Angular` 提供了服务人员支持。

`Preact` 是用于开发前端或用户界面的库。`Preact` 重量轻，大小为 `3kB`，是拥有相同 `ES6 API` 的 `ReactJS` 的替代产品。

**特性:**

*   它是 `ReactJS`。
*   这个轻量级库提供了 `React JS` 的大部分功能。
*   体积仍然小得多，它能更快、更高效地渲染数据。

## Preact 与 Angular 的区别

| 序号 | Preact | Angular |
| :--- | :--- | :--- |
| 1 | 它是 `ReactJS` 的轻量级替代品。 | 是一个用于开发前端应用程序的复杂框架。 |
| 2 | 它是用 `JavaScript` 编写的。 | 完全基于 `Typescript` 的单页面应用。 |
| 3 | 它体积小，约 `3kB`，但仍提供相同的响应式应用程序编程接口用于开发。 | 而 `Angular` 是一个重量级的 `MVC` 框架，适合开发大规模应用程序。 |
| 4 | 它使用并更新虚拟 `DOM`。 | 它匹配并更新真实 `DOM`。 |
| 5 | 它是一个 `JavaScript` 库，是 `ReactJS` 的轻量级替代品。 | 是一个用于开发单页面应用程序的复杂 `MVC` 框架。 |
| 6 | 安装命令：`npm install -g preact-cli` | 安装命令：`npm install -g @angular/cli` |
| 7 | 它是基于文档对象模型的架构。 | |
| 8 | 有效的内存使用 | `Angular` 使用的内存比 `Preact` 大。 |
