# GWT 和 React 的区别

> 原文: [https://www.geeksforgeeks.org/difference-between-gwt-and-react/](https://www.geeksforgeeks.org/difference-between-gwt-and-react/)

`GWT` 是一个开发工具包，用于构建和优化复杂的基于浏览器的应用程序。它的目标是实现高性能 web 应用程序的高效开发，而开发人员不必是浏览器怪癖、`XMLHttpRequest` 和 `JavaScript` 方面的专家。它是开源的，完全免费的，全世界成千上万的开发者都在使用它。

## GWT 特色

*   `GWT` 允许轻松地与 `Junit` 和 `Maven` 集成。
*   `GWT` 生成优化的 `JavaScript` 代码。
*   `GWT` 会自动为每个浏览器生成合适的 `JavaScript` 代码。
*   `GWT` 是一个用于构建大规模高性能 web 应用程序的框架。

`ReactJS` 是一个声明性的、动态的、可扩展的 `JavaScript` 库，用于构建应用程序的前端或用户界面。它是一个开源的、基于组件的前端库，只负责 web 应用和移动应用的视图层。它由脸书提供和管理。

## ReactJS 的特点

*   由于其自适应结构和可扩展性，它适用于大规模应用程序。
*   来自世界各地的开发者正试图添加更多功能。
*   它使开发者在处理项目时可以重用不同级别的代码组件。

## GWT 与 React 的区别

| **React** | **GWT** |
| --- | --- |
| `React` 具有动态类型。 | 没有动态类型。 |
| `React` 是一个用于构建用户界面的 `JavaScript` 库。 | `GWT` 是一个用于开发 web 应用程序的开发工具包。 |
| `React` 中复杂的架构使得传统方法难以跟上。 | 实时数据流处理很容易。 |
| `React` 由 Jordan Walker 和 Dennis Popov 开发。 | `GWT` 由 `Google` 开发。 |
| `React` 的大部分开发由用户完成。 | `GWT` 用户非常少。 |
| 它在浏览器中进行 `API` 调用和处理数据。 | `GWT` 处理来自浏览器的请求和认证、进行数据库调用等等。 |
| `React` 以 `MVC` 作为其设计模式。 | `GWT` 有模型视图、演示者活动和位置。 |
| `React` 没有 `WYSIWYG` 编辑器。 | `GWT` 有 `WYSIWYG` 编辑器。 |
| `React` 基于 `JavaScript`。 | `GWT` 基于 `Java` 编程语言。 |
