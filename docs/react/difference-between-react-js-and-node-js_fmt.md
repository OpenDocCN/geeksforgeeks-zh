# `React.js` 和 `Node.js` 的区别

> 原文：[https://www.geeksforgeeks.org/difference-between-react-js-and-node-js/](https://www.geeksforgeeks.org/difference-between-react-js-and-node-js/)

1.  `React.js`：
    这是 Facebook 在 2013 年开发的 JavaScript 库。这个库是为了改进和增强网络应用程序的用户界面而开发的，但是随着时间的推移，它已经发展了很多。它是开源的，支持不同的内置功能和模块，如表单模块、路由模块等。`ReactJS` 也用于制作单页网络应用程序。

2.  `Node.js`：
    这是一种服务器端语言，使用 JavaScript 运行时环境来执行要在浏览器外部执行的 JavaScript 代码。它是开源的，通过 `NPM` 支持很多模块，叫做节点包管理器。`Node.js` 是单线程的，它使用承诺，因此速度更快，更健壮。

## `React.js` 和 `Node.js` 的区别

| 序号 | `ReactJS` | `Node.js` |
| :--- | :--- | :--- |
| 1. | 由 Facebook 开发。 | 由 Ryan Dahl 开发。 |
| 2. | 发布于 2013 年。 | 发布于 2009 年。 |
| 3. | 是一个用于开发 UI 的库。 | 这是一个 JavaScript 运行时环境，帮助在浏览器外执行 JavaScript。 |
| 4. | 基本上用于客户端。 | 用于服务器端开发、客户端开发和 JavaScript。 |
| 5. | 用纯 JavaScript 编写。 | 用 `C`、`C++`、`JavaScript` 编写。 |
| 6. | 用于开发单页应用程序。 | 可以使用 `http` 或 `https` 模块创建可扩展的 HTTP 服务器。 |
| 7. | 引入了 `React Native` 以支持 `Android` 和 `iOS`。 | 仅支持 Web。 |
| 8. | 没有微服务和 API 支持。 | 你可以用 `Node.js` 创建微服务和 API。 |
| 9. | 对前端有很好的支持。 | 对前端的支持不多。 |
| 10. | 它有助于创建简单的 UI 测试用例。 | 支持 `Jasmine` 的服务器端测试用例。 |
