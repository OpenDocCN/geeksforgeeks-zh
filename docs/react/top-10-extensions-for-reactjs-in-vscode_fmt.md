# VSCode 中 ReactJS 的前 10 个扩展

> 原文：[https://www.geeksforgeeks.org/top-10-extensions-for-reactjs-in-vscode/](https://www.geeksforgeeks.org/top-10-extensions-for-reactjs-in-vscode/)

为了让你的开发更快，生活更轻松，安装一些如下图所示的 Visual Studio Code 扩展，为你的开发创造大量的动力工具。

## VSCode 中 ReactJs 的前 10 个扩展是：

*   Bracket Pair Colorizer 扩展。
*   Change Case 扩展。
*   Code Spell Checker 扩展。
*   Duplicate Selection 扩展。
*   EditorConfig for VS Code 扩展。
*   VSCode React Refactor 扩展。
*   NPM Intellisense 扩展。
*   ESLint 扩展。
*   ES7 React/Redux/GraphQL/React-Native Snippets 扩展。
*   Prettier - Code formatter 扩展。

### Bracket Pair Colorizer
这个扩展允许用颜色来识别匹配的括号。用户将勾勒出要匹配的字符，以及要使用的颜色。这个扩展的主要目的是允许匹配的括号用颜色来标识。

### Change Case
它是 Visual Studio Code 的 Node Change Case 的包装器。主要的工作是快速修改这个选择的情况或者当前的字。它也适用于多光标。

### Code Spell Checker
它是一个基本的拼写检查器，与 `camelCase` 代码配合良好。它基本上有助于捕捉常见的拼写错误，而将的误报数保持在较低水平。

### Duplicate Selection
这个扩展带来了指定文本的类似 Sublime 的重复，它内联复制它，而不是在打印操作期间，就像 VSCode 的默认行为一样。

### EditorConfig for VS Code
这个扩展带来了 `EditorConfig` 和 `.editorconfig` 支持到我们的 VSCode。该插件使用 `.editorconfig` 文件中的设置覆盖用户或工作区设置。

### VSCode React Refactor
这个简单的扩展为 React 开发者提供了重构代码的动作。它将 `JSX` 元素提取到文件或函数中。它还支持 `TypeScript` 和 `TSX` 语法。这个扩展很适合类、函数和箭头函数。

### NPM Intellisense
这是一个将自动完成所有 `NPM` 模块导入语句的插件。`NPM` 模块的所有导入自动得到通过此扩展处理。

### ESLint
它将 `ESLint` 集成到你的 VS Code 系统中。`ESLint` 静态分析你的代码，以便快速发现问题。

### ES7 React/Redux/GraphQL/React-Native Snippets
此扩展有助于为您的 VS 代码提供 `ES7` 中的 `JavaScript` 和 `React/Redux` 相关的片段以及 `Babel` 插件选项。

### Prettier - Code formatter
它是一个代码格式化的工具。这个 VS Code 包用来格式化你的 `JavaScript`/`TypeScript`/`CSS`。`Prettier` 它基本上解析你的代码，用自己的规则重新打印。
