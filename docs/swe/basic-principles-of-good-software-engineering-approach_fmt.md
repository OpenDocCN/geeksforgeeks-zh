# 良好软件工程方法的基本原则

> 原文：[https://www.geeksforgeeks.org/basic-principles-of-good-software-engineering-approach/](https://www.geeksforgeeks.org/basic-principles-of-good-software-engineering-approach/)

## 软件工程简介

[软件工程](https://www.geeksforgeeks.org/software-engineering-introduction-to-software-engineering/)是软件产品/应用开发的系统工程方法。它是与分析用户需求、软件产品的设计、开发、测试和维护相关的工程分支。

好的软件工程的一些基本原则是：

### 基本原则

1.  软件工程的基本原则之一是更好的需求分析，这能为项目提供清晰的愿景。最终，对用户需求的良好理解能够通过交付满足用户需求的良好软件产品来为其用户创造价值。
2.  所有设计和实现都应尽可能简单，这意味着应遵循`KISS`（Keep it Simple, Stupid）原则。它使代码变得非常简单，从而使调试和进一步维护变得简单。
3.  在整个开发过程中保持项目的愿景是软件项目成功最重要的事情。对项目的清晰愿景能引导项目以正确的方式发展。
4.  软件项目包含许多功能，所有功能都应以模块化方式开发，这样开发会更快、更容易。这种模块化使功能或系统组件相互独立。
5.  关注点分离原则的另一个特化是抽象，它用于抑制复杂的事物并向客户/用户提供简洁性，这意味着它只提供用户实际需要的东西，并隐藏不必要的东西。
6.  “先思考后行动”是软件工程必须遵循的原则，意味着在开始开发功能之前，首先需要思考应用架构，因为对项目开发流程的良好规划会产生更好的结果。
7.  有时开发人员会把所有功能堆砌在一起，但后来发现没有用。因此，遵循“不添加额外功能”的原则很重要，因为它只实现实际需要的东西，后续再按需实现其他功能，从而节省精力和时间。
8.  当其他开发人员在别人的代码中工作时，他们不应该感到惊讶，也不应该浪费时间去理解代码。因此，在必要的步骤提供良好的文档是开发软件项目的好方法。
9.  应遵循`Demeter`法则，因为它使类独立于其功能，并减少了类之间的连接和相互依赖性，这被称为耦合。
10. 开发人员应以满足通用性原则的方式开发项目，这意味着它不应局限于某些情况/功能，而应摆脱不自然的限制，并能够以广泛的方式向客户提供他们实际需要或普遍需要的服务。
11. 一致性原则在编码风格和设计图形用户界面（`GUI`）时很重要，因为一致的编码风格使代码更易于阅读，而`GUI`的一致性使用户在处理界面和使用软件时更容易学习。
12. 如果需要的任何东西已经存在，就不要浪费时间，利用开源并根据需求以自己的方式集成它。
13. 执行持续验证有助于检查软件系统是否符合需求规格说明并实现其预期目的，这有助于更好的软件质量控制。
14. 为了在当前技术市场趋势中立足，使用现代编程实践对于以最新和先进的方式满足用户需求非常重要。
15. 应保持软件工程的可扩展性，以增长和管理对软件应用程序不断增长的需求。
