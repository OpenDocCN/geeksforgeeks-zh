# 计划评审技术计划流程

> 原文：[https://www.geeksforgeeks.org/pert-planning-process/](https://www.geeksforgeeks.org/pert-planning-process/)

[项目评估和评审技术(`PERT`)](https://www.geeksforgeeks.org/project-evaluation-and-review-technique-pert/) 通过网络图描述活动和活动或任务的时间表。`PERT` 用于估计项目的完成时间。

计划包括以下步骤：

1.  **确定明确的活动和突破点：**
    将完成项目所需的所有任务和突破点列在表格中是方便的。突破点代表一个或多个活动的起点和截止日期。通过活动识别，后期阶段的任务可以扩展，以建立关于顺序和持续时间的信息。

2.  **确定活动的适当顺序：**
    `PERT` 计划过程中的此步骤用于规定活动的顺序。活动可以是串行或并行的，顺序代表一个活动对其他活动的依赖关系。正如第一步完成了活动的识别，在此步骤中识别活动之间的关系是必要的。分析活动执行的确切顺序是其他任务所要求的。

3.  **构建网络图：**
    借助活动识别和活动顺序信息绘制网络图。在网络图中，活动用带箭头的线表示，突破点用圆圈表示。为了将表格形式的活动信息转换为网络图，可以使用各种 `CASE` 工具，如 `MS-PROJECT`，这些工具简化了转换过程。

4.  **估算每项活动所需的时间：**
    项目中的所有活动都以持久或稳定的时间单位来说明。`PERT` 有能力处理活动完成时间中的模糊性和不可预测性。软件工程中用于活动完成时间的常用单位是周。

5.  **确定关键路径：**
    关键路径是花费时间最长的路径。网络图中有多条路径，展示了项目的开始和结束事件。关键路径是通过将每个序列中活动所花费的时间相加，然后确定项目中最长的路径来确定的。项目所需的总日历时间由关键路径给出。

6.  **`PERT` 图表的更新：**
    `PERT` 图表随着项目工作的进展而修改。随着项目工作的继续，对 `PERT` 图表进行了许多更改，例如增加资源和根据实际时间更改估计时间。
