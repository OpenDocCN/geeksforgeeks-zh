# 信息系统安全负责人：安全系统开发生命周期

> 原文：[https://www.geeksforgeeks.org/principal-of-information-system-security-security-system-development-life-cycle/](https://www.geeksforgeeks.org/principal-of-information-system-security-security-system-development-life-cycle/)

**安全系统开发生命周期**（`SecSDLC`）定义为在[软件开发周期（`SDLC`）](https://www.geeksforgeeks.org/software-engineering-sdlc-v-model/)中按顺序执行的一组程序。它的设计可以帮助开发人员创建软件和应用程序，从一开始就大大降低后期的安全风险。

安全系统开发生命周期（`SecSDLC`）与软件开发生命周期（`SDLC`）有些相同，但它们在周期的每个阶段执行的活动方面有所不同。`SecSDLC`消除了安全漏洞，其流程包括识别某些威胁和这些威胁在系统上代表的风险，以及实施必要的安全控制来应对、消除和管理相关风险。而在软件开发生命周期过程中，重点主要是信息系统的设计和实现。

## `SecSDLC`涉及的相位为：

### 系统调查
该流程由在组织最高管理层工作的官员/指令启动。为了执行这个过程，项目的目标和目的被优先考虑。定义了一个信息安全策略，它包含了安全应用程序和程序的描述，以及它们在组织系统中的实现。

### 系统分析
在此阶段，会对来自系统调查阶段的文档进行详细的文档分析。分析现有的安全策略、应用程序和软件，以检查系统中的不同缺陷和漏洞。同时分析未来可能出现的威胁可能性。风险管理也属于此过程。

### 逻辑设计
逻辑设计阶段涉及开发工具和后续蓝图，这些工具和蓝图涉及各种信息安全策略、其应用程序和软件。还起草了备份和恢复策略，以防止未来的损失。在发生任何灾难时，也会规划业务中应采取的步骤。在此阶段决定是否将公司项目外包。分析项目是否可以在公司内部完成，或者是否需要发送给另一家公司以完成特定任务。

### 物理设计
技术团队获取实施系统安全软件和应用程序所需的工具和蓝图。在此阶段，会调查不同的解决方案以应对未来可能遇到的任何不可预见的问题，对它们进行分析并记录下来，以覆盖在分析阶段遗漏的大部分漏洞。

### 实施
早期阶段决定的解决方案最终确定，无论项目是内部开发还是外包，都会提供适当的产品文档，以满足项目指定的要求。在各种团队的帮助下，积极进行项目的实施和集成过程，测试产品是否符合系统文档中指定的系统要求。

### 维护
在安全程序实施后，必须确保其正常运行并得到相应管理。安全程序必须相应地保持最新，以应对设计时可能未发现的新威胁。

这些是 `SecSDLC` 周期中涉及的步骤及其简要描述。
