# 软件定义存储(SDS)的俯视图

> 原文：[https://www.geeksforgeeks.org/over-view-of-software-defined-storagesds/](https://www.geeksforgeeks.org/over-view-of-software-defined-storagesds/)

**软件定义的存储:**

随着数据的生成呈指数级增长，存储数据和管理服务器变得更加昂贵和复杂。有几个系统可以帮助组织管理他们的数据。我们即将看到一个比传统系统更高效的系统。这就是软件定义存储。

那么，让我们来理解**SDS 是什么意思，为什么它更好**。

*   `Software-defined storage (SDS)` 是一种将存储软件与其硬件虚拟分离的方法。存储软件是物理存储和数据请求之间的一层，可帮助您控制存储请求。这意味着我们可以控制数据的存储位置和方式。
*   `SDS` 通常设计为在配备 `Intel X86` 处理器的基本服务器硬件上运行，与传统系统相比可以节省成本。

## 为什么 SDS 比传统系统好？

`SDS` 比传统系统更好，因为它实际上将存储软件从硬件中分离或抽象出来。这使您能够扩展存储容量，而不是添加另一个硬件。`SDS` 控制器软件提供连接、网络和存储访问设施。

让我们来看看 `SDS` 的关键特性。

1.  **自动化：** 通过简单高效的管理，减少手动流程和运营成本。
2.  **标准基础设施：** 为了管理和维护大量的存储设备和软件，需要一个 `API`。
3.  **可扩展性和透明度：** 能够扩展存储容量并监控其资源可用性和成本估算，而不会降低性能。

## 软件定义存储的工作方式:

这就是接下来要做的。`SDS` 从硬件中提取存储软件，并将所有存储软件和资源虚拟分组。这提供了一个由独立硬件组成的大型联合存储池。因此，您可以在其中的任何硬件上存储数据。它为您提供了根据需要升级或降级硬件的灵活性。

## SDS 的优势:

*   灵活性
*   可扩展性
*   敏捷性
*   自动化
*   成本效益
*   虚拟化和互操作性

1.  **灵活性：** `SDS` 基础设施可以构建在配备 `X86` 处理器的通用服务器上。因此，您不必担心选择不购买硬件的公司的 `SDS` 控制器。
2.  **可扩展性：** 更容易向虚拟池中添加额外的存储设备、`CPU` 和内存，以提高性能和容量。
3.  **能力：** `SDS` 可以同时支持传统和新一代应用程序。
4.  **自动化：** `SDS` 可以适应数据和性能要求，无需管理员进行任何手动操作。
5.  **成本效益：** 因为 `SDS` 高效利用可用资源并优化存储容量。`SDS` 的自动化通过自动化管理降低了运营成本。
6.  **虚拟化和互操作性：** `SDS` 使得虚拟整合各种存储资源并将其作为单个单元进行管理成为可能。通过 `SDS` 集成来自不同供应商的服务器硬件是可行的，因为它充当协调器来统一不同的存储服务并将其虚拟地放入一个池中。

有了这些好处，`SDS` 可以成为组织智能管理和存储数据的游戏规则改变者。
