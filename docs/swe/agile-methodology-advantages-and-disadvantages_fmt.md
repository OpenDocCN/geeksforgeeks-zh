# 敏捷方法的优缺点

> 原文：[`https://www.geeksforgeeks.org/agile-methodology-advantages-and-disadvantages/`](https://www.geeksforgeeks.org/agile-methodology-advantages-and-disadvantages/)

[敏捷软件开发方法论](https://www.geeksforgeeks.org/software-engineering-agile-software-development/)是软件开发的一个过程（如其他软件开发方法论——瀑布模型、V 模型、迭代模型等），但是，敏捷开发模型也是一种增量模型。软件以递增的、快速的周期发展。在英语中，敏捷的意思是“快速轻松地行动的能力”和快速响应变化的能力——这是敏捷软件开发的一个重要方面。

## 敏捷方法论的优势

1.  在敏捷方法中，软件的交付是持续不断的。
2.  客户感到满意，因为软件的每个冲刺（Sprint）工作特性都会交付给他们。
3.  客户可以看到符合他们期望的工作特性。
4.  如果客户对功能有任何反馈或变更，可以在当前产品版本中进行调整。
5.  在敏捷方法中，业务人员和开发人员需要每日互动。
6.  在这种方法中，注重产品的良好设计。
7.  即使在开发的后期阶段，需求的变更也是可接受的。
8.  敏捷/Scrum方法可以通过打破组织壁垒，并围绕组织目标培养信任与合作精神，来提高组织的协同效应。

## 敏捷方法的缺点

1.  在敏捷方法中，文档较少。
2.  有时在敏捷方法中，需求不是很明确，因此很难预测预期结果。
3.  在少数项目中，在软件开发生命周期开始时，很难估计所需的实际工作量。
4.  由于功能在不断演变，项目存在永远持续下去的风险。
5.  对于复杂的项目，很难估计资源需求和工作量。
