# 了解应用程序性能管理(APM)

> 原文：[https://www.geeksforgeeks.org/understanding-application-performance-management-apm/](https://www.geeksforgeeks.org/understanding-application-performance-management-apm/)

每天，我们以使用一些[软件应用程序](https://www.geeksforgeeks.org/application-software/)开始和结束一天。人们现在越来越忙，越来越多的时间都在关注效率。这就是为什么他们希望使用的软件应该是快速的，并且能够向用户提供实际需要的服务。

想想这种情况：我们正在打开一个软件应用程序，它花费了太多的时间加载；我们正在做一些活动，但是应用程序太慢；在这两者之间缺乏响应，我们需要等待对我们的请求的响应，我们没有得到我们想要的实际服务等等……在这里，性能问题就凸显出来了。

每个人都希望他们正在使用的软件应用程序能够提供高性能。公司或服务提供商也没有错过这一点。他们正在努力确保应用程序的良好性能。这就是为什么他们使用应用程序性能管理软件来管理和监控应用程序的性能。

## 应用性能管理(APM)

APM 也称为应用程序性能管理，它是一种软件，帮助组织在问题影响个人或商业业务以及最终用户之前预防和预测软件相关问题。

*   APM 软件帮助组织致力于满足用户对性能可用性的期望。
*   它执行应用程序的性能测量，并在软件执行任务遇到问题时向管理员发出警报，这些问题通常会自动引发许多其他问题。
*   APM 还负责应用程序性能监控。

## APM 的工作

APM 收集性能数据，并对其进行分析，以检测软件应用程序的潜在性能问题，并提供信息以采取措施解决这些问题。使用 APM 解决方案，企业可以满足其 IT 环境的标准性能，发现 bug 和潜在问题，并通过密切监控 IT 资源提供无限的用户体验。

如今，APM 为 IT 团队提供了他们需要的信息，以将应用程序的性能与业务结果联系起来，并且它还在问题影响最终用户之前识别性能问题，让 IT 团队了解性能问题产生的原因。

基本的区别在于它们如何收集信息和分析数据，如何让开发人员理解这些信息，以及 APM 对他们的帮助有多大。

在当今的数字领域，速度就是一切，尤其是在应用和网站领域。最终用户评估的是它有多么简单和快速。当应用程序变慢时，将直接影响业务、面向业务的事务和最终用户的期望。为了克服这类问题，APM 正在发挥作用。

### 不同的代理监控应用程序环境

不同的代理监控应用程序环境，并通过管理所有这些来支持基础架构。不同的代理包括：

1.  **数字体验监控：** 它收集性能数据，如加载时间、正常运行时间、停机时间等。响应时间意味着加载页面需要多长时间以及应用程序首次响应的速度。APM 从最终用户设备的用户界面进行监控。真实用户监控通常支持数字体验监控。
2.  **应用程序监控：** 应用程序监控包括应用程序框架（`.NET`、`C`、`C++`、`Java`）、数据库、`API`、中间件、Web 应用、`UI`、服务器及其监控的 `CPU` 利用率、网络性能和磁盘空间。堆栈监控管理代码级追踪。
3.  **数据库监控：** 数据库监控是针对 `SQL` 查询等性能的，它管理应用程序数据，也由应用程序监控代理提供。
4.  **可用性监控：** 它仅监控硬件组件和应用程序程序的实际可用性。作为一个应用程序程序，它将生成性能数据，但这些数据可能无法被最终用户访问。

## APM 的好处

*   减少停机时间。
*   为客户/最终用户提供更好的服务。
*   确保更快的创新。
*   提高客户满意度。
*   改善最终用户体验。
*   管理 `IT` 成本。
*   更好地了解您的业务。
