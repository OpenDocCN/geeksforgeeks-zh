# 了解 DMADV

> 原文：[https://www.geeksforgeeks.org/understanding-dmadv/](https://www.geeksforgeeks.org/understanding-dmadv/)

前提条件：[软件工程六适马](https://www.geeksforgeeks.org/six-sigma-in-software-engineering/)

## `DMADV`：

`DMADV` 是一个被定义为摩托罗拉的过程，其中包括六西格玛管理哲学。该术语代表定义、测量、分析、设计、验证它们在特定的有序过程中工作。`DMADV` 是杂类，属于未分类的子类。

这些包括六西格玛工作与极其雄心勃勃的目标，以尽量减少产品和服务中出现的缺陷。将制造过程中的改进与管理客户服务的改进结合起来，有助于在市场上提供更完整、更有利可图的产品。

## 让我们总结一下六西格玛规则：

1.  **Definition**：它陈述问题，指定客户，确定目标，并完成目标概要流程。这是框架的第一个基本阶段，在此阶段分配和识别目标。公司和利益相关者设定现实且可衡量的目标。定义概念后，项目策略会根据客户进行调整。
2.  **Measurement**：它根据数量确定参数，在最佳测量模式下工作，收集数据并进行实验。考虑关键因素，并以能够评估质量、过程能力及所有可能风险因素的方式定义需求度量。
3.  **Analysis**：确定性能目标和影响输出过程的输入过程。它接近于开发设计替代方案，定义更好的需求组合以在多个约束条件下实现价值，进行设计并最终找到最佳产品。
4.  **Design**：它处理细节，优先考虑高级设计，优化原型模型，检测错误并运行模拟。它处理高级设计，开发更详细的设计和更高功能的原型。
5.  **Verification**：确认期望，扩大部署，确认经验并进行测试。团队的最后阶段是验证设计，使其满足要求并可接受。这也是整个计划中的重要一步，规定了从过程到正常运行的过渡。

这些步骤仔细检查现有的过程，因此被称为西格玛，它不符合六西格玛要求。所有这些西格玛过程都是由六西格玛执行的。

它定义了与服务或产品相关的客户需求。另一方面，它测量客户的规格和需求。`DMADV` 开发了一个合适的商业模式来满足客户的要求。

## `DMADV` 遵循的策略：

*   提高效率和效果
*   不可互换的设计
*   根据客户需求和服务改进六西格玛规则
*   优先级
*   根据产品服务分析和评估过程设计
*   检查结果并保持性能

## `DMADV` 优势：

1.  最大化客户满意度。
2.  最小化缺陷率。

## `DMADV` 的劣势：

1.  对员工进行六西格玛培训耗时且有时成本较低。
2.  优先考虑创新，而创造力是主要缺点。
