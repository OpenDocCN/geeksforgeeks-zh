# 软件工程|案例的好处

> 原文：`https://www.geeksforgeeks.org/software-engineering-benefits-of-case/`

使用 [CASE](https://www.geeksforgeeks.org/computer-aided-software-engineering-case/) 设置或单独的 CASE 工具会带来一些好处。其中一些好处是：

*   使用 CASE 环境产生的一个关键好处是在所有开发阶段节省成本。不同的研究表明，理解 CASE 的影响可以将问题减少 30% 到 40%。
*   CASE 工具的使用带来了质量的显著提升。这主要是因为人们可以轻松地重述代码开发的所有阶段，因此人为错误的可能性大大降低。
*   CASE 工具有助于制作高质量且一致的文档。由于与产品相关的必要信息存储在一个非常集中的存储库中，存储信息的冗余减少，因此文档不一致的可能性也大大降低。
*   CASE 承担了极端代码工程师的大部分工作。例如，他们不必仔细检查 DFD 的级别，而是可以通过按一下按钮轻松完成。
*   CASE 具有革命性，可以在代码维护中实现显著的成本节省。这不仅是因为 CASE 设置在可追溯性和一致性检查方面具有巨大价值，还因为遵循 CASE 设置在代码开发的每个阶段捕获的系统信息。
*   CASE 的引入对组织的操作设计产生了影响，使其面向结构化和有序的方法。
