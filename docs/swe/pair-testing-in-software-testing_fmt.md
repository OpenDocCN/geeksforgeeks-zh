# 软件测试中的配对测试

> 原文：[https://www.geeksforgeeks.org/pair-testing-in-software-testing/](https://www.geeksforgeeks.org/pair-testing-in-software-testing/)

前提条件：[软件测试](https://www.geeksforgeeks.org/software-testing-basics/)

## 配对测试

结对测试是由在机器后面操作的两个团队成员在软件中进行的验证。第一个成员控制鼠标和键盘，第二个成员做笔记，讨论测试场景和准备/提问。其中一个必须是测试人员，下一个必须是开发人员或业务分析师。

[结对编程](https://www.geeksforgeeks.org/pair-programming/#:~:text=Pair%20programming%20is%20a%20development,may%20switch%20their%20role%20frequently.)是极限编程中常见的做法。因此，结对编程被认为是一种很好的软件编程方法。像明智对测试是一个类似的软件测试过程。

## 使用结对测试的优势

1.  开发人员从不同或积极的视角看待软件。他们讨论并找出如果我执行它会发生什么，或者如果业务分析师不执行它会发生什么。
2.  当与业务分析师一起进行配对测试时，他们会在分析师和测试人员之间交流想法和知识。
3.  当一个新项目由新团队成员启动时，测试人员和开发人员之间经常存在障碍。
4.  如果我们发现任何问题并想在错误登记系统中登记，这些问题将被自动纠正，因此两人一组行走将有助于互相完善。

## 使用配对测试的缺点

当你不知道配对测试设置的准确条件时，你应该或不应该使用它。

1.  所有强制性测试都应该是自动化的，而配对测试的解决方案是发现而非测试用例。我们不能在测试自动化后立即使用配对测试会话的结果。
2.  当有团队成员时，两个团队成员可能会结束彼此的分类。这就是为什么当团队成员沟通或合作不佳时，我们不应该使用配对测试。
3.  如果你计划实现一个结构化的测试用例，一起执行测试用例可能无法增加更多价值或零价值。这个任务应该由一个团队成员单独完成。

## 结对测试设置

1.  团队成员应该一起预订。当我们试图强制合作时，它不起作用，所以我们需要创造一个可行的氛围。
2.  应该有一个单独的房间或桌子，团队成员可以在不受干扰的情况下进行。为了更好地工作，他们还应该关闭手机和通知。
3.  工作场所由坐在桌子后面的两个人组成。你不能强迫没有空间的人。

## 准备结对测试会话

1.  我们应该创建一个探索性测试章程。
2.  我们应该明确测试的重点和大纲。
3.  我们应该指定测试的目的或目标。
4.  我们应该为测试设定一个时间限制。通常一节课是90分钟。
5.  我们应该以适当的方式计划会议。

## 配对测试会话的执行

*   在会话期间，团队成员应讨论测试的部分和测试的深度。测试应具有目标、重点和测试部分，如探索性测试章程所述。
*   第一组成员（驾驶员）控制键盘和鼠标，而第二组成员分析、提问并准备笔记。

## 结对测试整理

配对测试会话完成后，发现的问题将提交给错误登记系统。如果需要而没有完成，将升级探索性测试章程，在发现问题的地方测试目标，并检查其他备注。探索性测试章程的范围是对测试时期的简短评估，包括什么进展顺利，什么需要改进。

## 最后

它是团队工作和测试的融合，但是它有很多优点，比如分享关于测试和被测系统的知识，培训新成员，在成员之间制造障碍等等，这很有趣。我们应该明智地使用配对测试。
