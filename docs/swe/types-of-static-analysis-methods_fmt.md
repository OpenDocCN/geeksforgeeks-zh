# 静态分析方法的类型

> 原文：[`https://www.geeksforgeeks.org/types-of-static-analysis-methods/`](https://www.geeksforgeeks.org/types-of-static-analysis-methods/)

## 软件的静态分析

静态分析涉及一组用于分析软件源代码或目标代码的方法，以确定软件如何运行，并建立检查其正确性的标准。静态分析研究源代码而不执行它，并揭示各种各样的信息，如所用模型的结构、数据和控制流、语法准确性等。

静态分析方法有几种类型：

### 1. 控制分析

该软件侧重于检查调用结构、控制流分析和状态转换分析中使用的控制。调用结构通过识别调用者和被调用者与模型相关。调用结构可以是进程、子程序、函数或方法。`Control flow analysis`检查控制转移的序列。此外，它还检查模型中的低效构造。会创建一个模型图，其中条件分支和模型连接点由节点表示。

### 2. 数据分析

确保对数据对象（如数据结构和链表）应用了正确的操作。此外，此方法还确保正确定义的数据被正确使用。数据分析涉及两种方法，即`Data dependency`和`Data flow analysis`。`Data dependency`对于评估多处理器间同步的准确性至关重要。`Data flow analysis`检查变量的定义和上下文。

### 3. 故障/失效分析

它分析模型中的故障（不正确的组件）和失效（模型组件的不正确行为）。此方法使用输入-输出转换描述来识别导致失效的条件。为了确定特定条件下的失效，会检查模型设计规范。

### 4. 界面分析

该软件通过验证和验证交互以及分发模拟来检查代码。界面分析有两种基本技术：用户界面分析检查子模型界面并确定界面结构的准确性。用户界面分析检查用户界面模型，以及在用户与模型交互期间为防止错误而采取的预防措施。该方法还关注界面如何准确地集成到整体模型和仿真中。
