# 计算机网络中的服务器

> 原文：[https://www.geeksforgeeks.org/servers-in-computer-network/](https://www.geeksforgeeks.org/servers-in-computer-network/)

在计算中，`服务器`是计算机`程序或设备`，其为作为其他程序或设备的被叫`客户端`提供功能。这种架构被称为`客户端-服务器模型`。单个整体计算分布在多个进程或设备上。服务器可以提供各种称为服务的功能。这些服务包括在多个客户端之间共享数据或资源，或者为一个客户端执行计算。单个服务器可以服务多个客户端，单个客户端可以使用多个服务器。客户端进程可以在同一设备上运行。它还可以通过网络连接到服务器，在不同的设备上运行。服务器示例可以包括`数据库服务器`、`邮件服务器`、`打印服务器`、`文件服务器`、`网络服务器`、`应用服务器`和`游戏服务器`。

最常见的客户端-服务器系统是通过`请求-响应模型`实现的，即客户端向服务器发送请求。在这个模型中，服务器执行一些操作，并将响应发送回客户端，通常带有结果或确认。将计算机指定为服务器级硬件意味着它专用于在其上运行服务器。这意味着它比标准的个人电脑更强大、更可靠。但是大型计算集群可能由许多相对简单、可替换的服务器组件组成。

## 服务器类型及其应用

1.  **Application server –**
    这些服务器托管`Web应用程序`（在`Web浏览器`中运行的计算机程序），允许网络中的用户运行和使用它们，无需在自己的计算机上安装副本。这些服务器不必是`万维网`的一部分。它们的客户端是装有`Web浏览器`的计算机。
2.  **Catalog server –**
    这些服务器维护一个索引或目录表，其中包含可在大型分布式网络中找到的信息。分布式网络可能包括计算机、用户、在文件服务器上共享的文件以及`Web应用程序`。目录服务器的示例是`目录服务器`和`名称服务器`。它们的客户端是任何需要在网络上查找内容的计算机程序。例如，尝试登录的域成员、查找电子邮件地址的电子邮件客户端，或查找文件的用户。
3.  **Communications server –**
    这些服务器维护一个环境，供一个通信端点查找其他端点并与之通信。根据网络的开放性和安全参数，这些服务器可能包含也可能不包含通信端点目录和在线状态检测服务。它们的客户端是通信端点。
4.  **Computing server –**
    这些服务器通过网络共享大量的计算资源，包括`CPU`和`RAM`。任何需要比个人计算机所能提供的更多`CPU`能力和`RAM`的计算机程序都可以使用这类服务器。客户端必须是联网的计算机以实现`客户端-服务器模型`，这是必要的。
5.  **Database server –**
    这些服务器通过网络维护和共享任何形式的数据库。数据库是具有预定义属性的有组织的数据集合，可以以表格形式显示。这些服务器的客户端是电子表格、会计软件、资产管理软件或任何消耗结构化数据的计算机程序，尤其是大量数据时。
6.  **Fax server –**
    这些服务器通过网络共享一台或多台传真机，免去了物理访问的麻烦。任何传真发送者或接收者都是这些服务器的客户端。
7.  **File server –**
    通过网络共享文件和文件夹、存储文件和文件夹的空间，或两者兼有。联网的计算机是预期的客户端，尽管本地程序也可以是客户端。
8.  **Game server –**
    这些服务器使多台计算机或游戏设备能够玩多人游戏。个人计算机或游戏机是它们的客户端。
9.  **Mail server –**
    这些服务器使电子邮件通信成为可能，就像邮局使传统邮件通信成为可能一样。这些服务器的客户端是电子邮件的发送者和接收者。
10. **Print server –**
    这些服务器通过网络共享一台或多台打印机，免去了物理访问的麻烦。它们的客户端是需要打印东西的计算机。
11. **Proxy server –**
    这个服务器充当客户端和服务器之间的中介，接受来自客户端的传入流量并将其发送到服务器。使用代理服务器的原因包括内容控制和过滤、提高流量性能、防止未经授权的网络访问，或者仅仅是将流量路由通过一个庞大而复杂的网络。它们的客户端是任何联网的计算机。
12. **Web server –**
    这些服务器托管`网页`。`Web服务器`负责使`万维网`成为可能。每个网站都有一个或多个`Web服务器`。它们的客户端是装有`Web浏览器`的计算机。
