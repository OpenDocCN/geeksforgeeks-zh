# 物联网中的会话层消息协议

> 原文：[https://www.geeksforgeeks.org/session-layer-messaging-protocols-in-iot/](https://www.geeksforgeeks.org/session-layer-messaging-protocols-in-iot/)

先决条件–[物联网简介](https://www.geeksforgeeks.org/introduction-to-internet-of-things-iot-set-1/)

会话层通过控制发送方和接收方之间的数据来管理网络两个端点之间的连接，其中会话层协议负责物联网生态系统中数据的实际传输。这就是为什么这些会话层协议被称为物联网`消息协议`或有时被称为物联网`数据协议`。大多数物联网应用使用 `TCP` 和 `UDP` 进行传输，这些消息协议可以通过 [`TCP` 或 `UDP`](https://www.geeksforgeeks.org/differences-between-tcp-and-udp/) 运行。

不同的标准化组织有不同类型的消息传递协议，根据它们的实现方式，它们被使用。以下是物联网生态系统中使用的一些流行的物联网消息协议。

1.  **Message Queue Telemetry Transport (`MQTT`)：**
    `MQTT` 是最广泛采用的 M2M（机器对机器）消息传递协议，由 IBM 于 1999 年推出。它使用发布者-订阅者机制运行，即发布者是轻量级传感器，它们收集数据并连接到中介层，即代理（broker）来发送数据。然后订阅者是从代理收集所需传感数据的应用程序。这种机制最大限度地减少了有效载荷。它主要用于拥有大量低功耗小型设备的网络，或者物联网设备可能带宽有限且需要通过互联网进行监控或管理的场景。

2.  **Advanced Message Queuing Protocol (`AMQP`)：**
    `AMQP` 是一种开放标准的会话层协议，用于发送事务性消息，因此主要用于金融行业。它运行在 `TCP`（传输控制协议）或 `UDP`（用户数据报协议）之上，这与 `MQTT` 几乎相似。但唯一的区别是代理（Broker）包含交换机（Exchange）和队列（Queues），因此它专注于不丢失消息。交换机从发布者接收消息，并将它们分发到相应的队列。不同的订阅者从相应的主题队列中收集主题相关的传感数据。

3.  **Constrained Application Protocol (`CoAP`)：**
    `CoAP` 是一种使用 `RESTful` 架构的会话层协议，该架构是 `HTTP` 客户端和服务器之间的标准接口。它使用 `UDP`（用户数据报协议）协议进行轻量级实现，因此它基于两个子层，即用于交互的消息层和请求/响应层。该协议旨在解决基于 `HTTP` 协议的物联网系统。因此，`CoAP` 利用所有 `HTTP` 方法（如 `GET`、`PUT`、`PUSH`、`DELETE`）进行消息传输和访问资源。它是一种专门的 Web 传输协议，在物联网中用于在相同受限网络上的受限节点/设备。

4.  **Secure Message Queue Telemetry Transport (`SMQTT`)：**
    `SMQTT` 作为 `MQTT` 协议的扩展工作。它基于加密消息机制，因此提供了一种安全的消息传递标准。在该协议中，订阅者向所有节点发送加密消息，节点接收加密消息并在解密后使用消息。加密和解密活动是使用主密钥进行的。

    该协议遵循四个主要阶段，即设置、加密、发布、解密。

    1.  在设置阶段，发布者和订阅者都在代理附近注册自己并获取主密钥。
    2.  在加密阶段，代理对发布的消息进行加密。
    3.  在发布阶段，代理向订阅者提供加密数据。
    4.  在解密阶段，即最后一个阶段，用户使用该主密钥对数据/消息进行解密。

5.  **数据分发服务 (`DDS`)：**
    `DDS` 是另一种发布-订阅协议，但它不同于将它们连接到服务器的 `MQTT`，这里 `DDS` 协议是一种无代理架构，这就是为什么它是一种比 `MQTT` 更高速、更高性能的协议，因为它不依赖于任何中间系统。它是由对象管理组 (`OMG`) 为设备间通信而设计的。该协议有两个基本子层，即以数据为中心的发布-订阅层 (`DCPS`) 和数据本地重建层 (`DLRL`)。`DCPS` 层负责向用户传递消息，`DLRL` 层负责在应用层简单集成 `DDS`，但这是可选的。
