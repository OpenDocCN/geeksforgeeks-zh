# 计算机网络中的访问控制

> 原文：[https://www.geeksforgeeks.org/access-control-in-computer-network/](https://www.geeksforgeeks.org/access-control-in-computer-network/)

**访问控制**是一种限制对系统或物理或虚拟资源的访问的方法。这是一个用户可以访问系统、资源或信息并被授予某些特权的过程。访问控制是一种安全技术，可以控制谁可以查看不同的方面、可以查看什么以及谁可以使用计算环境中的资源。这是安全领域的一个基本概念，可以降低企业或组织的风险。

为了建立一个安全的系统，使用了电子访问控制系统，该系统依赖于用户证书、访问读卡器、审计和报告来跟踪员工对受限业务地点和区域的访问。这些系统包括访问控制面板，以禁止进入敏感区域，如警报和锁定区域，以防止未经授权的访问或操作。

访问控制系统通过评估所需的登录凭证来执行用户和实体的识别、认证和授权，登录凭证可能包括密码、`pin`、生物计量扫描或其他认证因素。存在需要两个或更多认证因素的多因素认证，这通常是保护访问控制系统的分层防御的重要部分。

## 认证因素

*   密码或个人识别码
*   生物计量测量（指纹和视网膜扫描）
*   卡片还是钥匙

根据法规遵从性要求和要保护的信息技术的安全级别，使用不同的访问控制模型。基本上访问控制有两种类型：

1.  **物理访问控制：**
    物理访问控制限制进入校园、建筑物、房间和物理 `IT` 资产。
2.  **逻辑访问控制：**
    逻辑访问控制限制与计算机网络、系统文件和数据的连接。

## 门禁模型

### Attribute-based Access Control (ABAC)
在此模型中，通过评估一组规则、策略和关系来授予或拒绝访问，这些规则、策略和关系使用用户、系统和环境条件的属性。

### Discretionary Access Control (DAC)
在 `DAC` 中，数据的所有者决定谁可以访问特定资源。

### History-Based Access Control (HBAC)
通过评估查询方的活动历史来授予或拒绝访问，该历史包括行为、请求之间的时间以及请求的内容。

### Identity-Based Access Control (IBAC)
使用此模型，网络管理员可以更有效地根据个人需求管理活动和访问。

### Mandatory Access Control (MAC)
一种控制模型，其中访问权限由中央机构根据多个安全级别进行监管。`Security Enhanced Linux` 在 `Linux` 操作系统上使用 `MAC` 实现。

### Organization-Based Access control (OrBAC)
此模型允许策略设计者独立于实现来定义安全策略。

### Role-Based Access Control (RBAC)
`RBAC` 允许基于职位进行访问。在提供对对象的访问时，`RBAC` 在很大程度上消除了自由裁量权。例如，人力资源专家不应该有创建网络账户的权限。

### Rule-Based Access Control (RAC)
`RAC` 方法在很大程度上是基于上下文的。例如，只允许学生在一天中的特定时间使用实验室。
