# JSF | Java 服务器面临

> 原文:[https://www.geeksforgeeks.org/jsf-java-server-faces/](https://www.geeksforgeeks.org/jsf-java-server-faces/)

`JSF` 技术包括一组应用编程接口，它们代表不同的用户界面组件，并帮助管理它们的状态。这些应用编程接口进一步帮助处理用户界面组件上的事件，并通过用户界面组件验证用户输入。`JSF` 框架提供了创建简单和复杂应用程序的灵活性，因为该技术使用最流行的 `Java` 服务器技术(`Servlet` 和 `Java Server Pages`)，并且不将开发人员限制在特定的标记语言或客户端设备上。与 `JSF` 应用编程接口捆绑的用户界面组件类包含各种组件功能的逻辑实现，并且没有任何特定于客户端的呈现逻辑。因此，`JSF` 用户界面组件可以为不同的客户端设备呈现。目前，`SF` 提供了一个自定义渲染器和 `Java Server Pages(JSP)` 自定义标记，用于为 `HTML` 客户端渲染 `UI` 组件。

`JSF` 是一个健壮的 `Web` 应用程序框架，它实现了一个事件编程模型来处理客户端在不同 `UI` 组件上执行的不同事件和操作。为了处理每个事件，应该在服务器端注册一个侦听器。开发 `Web` 应用程序时，开发人员必须在源代码中编写导航规则，以便从一个网页导航到另一个网页。`JSF` 提供了一种简单的方法来定义配置文件中的导航规则，并向客户端显示不同的错误消息，显示错误的真正原因。这些消息是在根据某些验证规则验证用户输入时生成的，可以显示在包含用户界面组件的同一页面上。

有不同的 `Web` 应用程序框架来实现一种或多种 `MVC` 设计模式。`JSF` 是基于 `MVC2` 模式的，这个模式是基于组件类型开发的。在这种模式中，开发人员只需专注于他们各自的组件，引入独立的层，如模型视图和控制器，并通过使网络应用程序易于维护来帮助开发人员专注于单一类型的组件。组件的不同类别(如模型视图和控制器)是为不同的功能(如使用文本字段、对话框简单标签和颜色选择器)创建的，可以单独使用

## 解读 JSF 的特色

`Java` 技术提供了各种框架来开发网络应用程序。其中一些框架，如 `Struts`，比 `JSF` 更受欢迎，但 `JSF` 丰富而简单的特性使其成为设计和管理网络应用程序中用户界面组件的首选之一。

以下是 `JSF` 的各种特性:

*   提供了一个易于使用的环境，即集成开发环境(`IDE`)，用于使用 `JSF UI` 组件开发 `Web` 应用程序。它拥有来自 `Sun`、`IBM` 和 `Oracle` 等公司的广泛工具支持。
*   通过其自身的一组标签，可以方便地在网页中创建复杂的用户界面模块。这些标签作为 `JSP` 自定义标签库提供。对于 `JSF`，设计用户界面模块很容易，因为它基于 `MVC` 设计模式，清晰地将表示与业务逻辑分离。
*   提供了管理网页中所有 `UI` 组件的方法。用户界面模块的管理包括验证用户输入、组件状态、页面导航和事件处理。
*   提供了一个可扩展的架构，这意味着您可以在 `JSF` 之上添加其他功能，并且可以轻松地自定义和重用 `JSF UI` 组件。
*   支持多种客户端设备。有不同的渲染器来为不同的客户端设备渲染或显示类似的用户界面模块。可以扩展各种组件类以创建自定义组件标签库，以支持特定类型的客户端。
*   包含支持国际化的组件，并支持根据指定的区域设置显示本地化消息。
*   支持标准的快速应用程序开发(`RAD`) `Java Web` 应用程序框架，通过一组可重用的组件。
*   可以快速开发强大的应用程序，它提供了一种方法让开发人员将视觉组件链接到控制器或模型组件而不破坏层次。
*   为 `JSF` 页面提供了一种表达式语言(`EL`)。因为 `JSF` 页面使用 `JSP` 标签，所以很难将单独的 `EL` 嵌入到 `JSF` 页面中。`Java` 规范的一个关键关注点是保持其不同 `Web` 层技术（如 `ISP`、`JSF` 和 `JSP` 标准标签库(`JSTL`)）的一致性。这个协议导致了统一 `EL` 的创建，它整合了 `JSP 2.0 EL` 和 `JSF 1.1 EL`。`JSP 2.1` 和 `JSF 1.2` 支持这个统一的 `EL`。换句话说，您可以在 `JSF` 组件中使用 `JSTL` 标签。
*   帮助使用异步 `JavaScript` 和 `XML` 技术(`AJAX`)构建 `Web 20` 应用程序，并进一步降低创建 `UI` 组件的复杂性。
*   使用 `faces-config.xml` 文件中的元素将应用程序范围的资源包配置为 `EL`。
